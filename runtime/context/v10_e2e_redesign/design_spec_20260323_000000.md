# Global E2E UX/UI Architecture & Design Specification

**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 10.0 (End-to-End Holistic Redesign)
**Date:** 2026-03-23
**Lead Architect:** Senior UX/UI Architect & Lead Designer
**Reference:** `Merkle-Brand-Guidelines-2025v1.0.pdf`

---

## 1. Architectural Vision: "AI-First Enterprise OS"

The previous iterations successfully built functional silos (Kanban, My Tasks, Analytics) and mapped them to the Merkle Brand. However, to truly compete with enterprise tools like Linear, Jira, and Asana, we must transition from "a collection of pages" to a **Holistic Enterprise Operating System**. 

**Core UX Principles for v10.0:**
1.  **Zero-Context-Switching:** Users should never have to leave the board to update a task or view a profile. We will heavily utilize `<Sheet>` (Right-side slide-overs) and `<Dialog>` components.
2.  **Keyboard-First Navigation:** Implement a global Command Palette (`Cmd + K`) for rapid navigation, task creation, and global search.
3.  **Information Density Control:** Provide "Compact" and "Comfortable" view toggles for data tables and Kanban boards to suit both PMOs (high density) and Developers (comfortable).
4.  **Brand-Immersive & Accessible:** Strictly adhere to `merkle-charcoal`, `merkle-red`, and `merkle-navy`. Ensure WCAG 2.1 AA contrast ratios across all interactive elements.

## 2. Global Layout & Shell Architecture

### 2.1 The Global Application Shell
*   **Navigation Hierarchy:** 
    *   **Left Sidebar (Collapsed by default on mobile, sticky on desktop):** 
        *   **Top:** Merkle Logo (clickable to `/`).
        *   **Workspace Section:** My Tasks (Inbox), Dashboard (Analytics), Directory.
        *   **Projects Section:** Accordion-style list of pinned projects.
        *   **Bottom:** User Profile dropdown & Global Settings.
    *   **Top App Bar (New in v10):**
        *   **Breadcrumbs:** Clear hierarchical context (e.g., `Workspace / Project Alpha / Sprint 2`).
        *   **Global Search:** A prominent Search Input that triggers the `Cmd + K` Command Palette.
        *   **Quick Actions:** A persistent `[ + Create Issue ]` button (Primary `merkle-red` color).

### 2.2 Command Palette (`Cmd + K` / `Ctrl + K`)
*   **Trigger:** Anywhere in the app.
*   **UI:** A centered `<Command>` dialog with a blurred backdrop (`backdrop-blur-sm bg-black/50`).
*   **Functions:** Search across Tasks, navigate to Projects, or type "Create..." to instantly open the New Task modal.

## 3. End-to-End User Journeys & View Specifications

### Journey 1: The PMO / Leadership View (Analytics & Portfolio)
*   **Path:** `/analytics`
*   **Layout Redesign:** 
    *   Transition from a simple grid to a **Bento-box layout**.
    *   **Top Row:** High-level KPIs (Total Active Projects, Avg Cycle Time, System Health).
    *   **Main Content:** A wide horizontal Bar Chart mapping Resource Workloads (using `merkle-red` and `merkle-navy` gradients).
    *   **Right Column:** "AI Insights" panel. The system automatically flags bottlenecks (e.g., "User A has 15 tasks in 'In Progress'"). This uses a subtle `bg-merkle-navy/10` card to denote AI-generated content.

### Journey 2: The Contributor View (My Tasks Inbox)
*   **Path:** `/` or `/my-tasks`
*   **Layout Redesign:** 
    *   Transition from a standard table to a **Split-Pane Layout** (similar to modern email clients).
    *   **Left Pane (30%):** A scrollable, compact list of tasks grouped by `Status` or `Due Date`.
    *   **Right Pane (70%):** When a task is clicked, its full details load instantly in the right pane. No modals required. This enables rapid triaging of the inbox.

### Journey 3: The Agile Execution View (Kanban Board)
*   **Path:** `/projects/{key}/boards`
*   **Layout Redesign:** 
    *   **Header Controls:** Add Filters ("Assigned to me", "High Priority") and a View Toggle (Kanban vs. List).
    *   **Columns:** Sticky headers so columns scroll independently. Column backgrounds use a very faint `bg-zinc-900/50` to delineate drop zones.
    *   **Task Cards:** 
        *   Make them visually lighter. Remove heavy borders, rely on subtle shadows (`shadow-sm`) and micro-interactions (lift on hover).
        *   Priority indicators change from dots to specific Material Icons (e.g., `keyboard_double_arrow_up` in `merkle-red` for High).
        *   Avatars overlap cleanly at the bottom right.

### Journey 4: The Holistic Task Modal (Collaboration)
*   **Trigger:** Clicking a task from the Board or Search.
*   **Layout Redesign:** Slide-over `<Sheet>` from the right edge of the screen instead of a center modal. This preserves the user's context of the board behind it.
*   **Content Structure:**
    *   **Header:** Task ID, Status Dropdown, and "Copy Link" button.
    *   **Left Column (Main):** Title (Large inline-editable text), Rich-text Description, Sub-tasks (Checklist), and the Activity/Comment Timeline.
    *   **Right Column (Sidebar):** Metadata (Assignee, Priority, Dates, Project, Custom Fields).
*   **Collaboration:** Comments feature @mentions, and AI Agent interactions are clearly bubbled with the `merkle-navy` brand styling.

## 4. Micro-Interactions & Animation Guidelines
To achieve a "premium" feel, animations must be purposeful and fast.
1.  **Transitions:** Standardize on `duration-200 ease-in-out` for hover states (buttons, table rows).
2.  **Slide-overs:** The Task `<Sheet>` should slide in with a subtle cubic-bezier curve to feel snappy but not abrupt.
3.  **Drag and Drop:** When dragging a Kanban card, the card should tilt slightly (`rotate-2`) and cast a larger shadow (`shadow-xl`) to indicate elevation.
4.  **Optimistic UI:** When a user changes a status or posts a comment, the UI must update instantly. Network requests happen in the background.
