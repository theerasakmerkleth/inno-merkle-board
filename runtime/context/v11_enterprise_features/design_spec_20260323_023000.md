## 1. UX Vision & Cognitive Load Management
Sprint 11 introduces high-density interactive elements (Rich Text, File Uploads, Checklists). To maintain our "Zero-Context-Switching" and "Holistic OS" mandates, these new components must feel natively integrated into the `<Sheet>` (Task Modal) without cluttering the interface.

**Design Principles:**
1.  **Progressive Disclosure:** Advanced formatting options and file dropzones should only be prominent when the user is actively engaging with them (e.g., focusing on the text area).
2.  **Feedback & Micro-interactions:** Real-time WebSocket updates (US-024) must be subtle but clear. A brief highlight/flash on a card that moved automatically helps the user track changes without being jarring.
3.  **Brand Alignment:** All new components (progress bars, notification badges) will strictly use `merkle-red` and `merkle-navy`.

## 2. Component Redesign & Wireframes

### 2.1 The Task Details `<Sheet>` (Epic-011 & Epic-012)
The right-side slide-over sheet is expanding to become a full collaboration hub.

*   **Rich Text Description (US-021):**
    *   **View Mode:** Rendered Markdown/HTML with clear typography (H1-H6, lists, inline code snippets in `bg-zinc-900`, blockquotes with `border-l-merkle-red`).
    *   **Edit Mode:** A minimalist WYSIWYG toolbar appears on focus (Bold, Italic, Strikethrough, Link, Bullet List, Numbered List, Code). 
    *   **Library Suggestion:** Use `TipTap` or `Quill` wrapped in a custom Tailwind container matching `bg-merkle-charcoal`.
*   **File Attachments Dropzone (US-022):**
    *   **UI:** A dedicated section below the description. The entire `<Sheet>` can act as a drop target when a file is dragged over it, showing a dashed border overlay (`border-2 border-dashed border-merkle-red bg-merkle-red/5`).
    *   **Preview Grid:** Uploaded files appear as a horizontal scrolling list of thumbnails (for images) or standard icons (for PDFs/Docs). Hovering shows a delete/download overlay.
*   **Sub-task Checklists (US-023):**
    *   **Location:** Between the Description and the Comments section.
    *   **UI:** 
        *   A simple input field: `[ + Add checklist item ]`.
        *   List of items with native HTML checkboxes or `shadcn/Checkbox`.
        *   **Progress Bar:** A thin horizontal bar (`h-1`) above the checklist showing completion percentage, filling up with `bg-merkle-navy`.

### 2.2 Global App Shell Updates

*   **In-App Notification Bell (US-025):**
    *   **Location:** Top App Bar, right-aligned next to the Command Palette search button.
    *   **Icon:** Material Icon `notifications`.
    *   **Badge:** A small `bg-merkle-red` dot (`w-2 h-2 rounded-full`) positioned at the top-right of the icon when unread notifications exist.
    *   **Dropdown Menu (`shadcn/DropdownMenu`):** Clicking reveals a panel with the last 5-10 notifications (e.g., "User A assigned TASK-123 to you"). Unread items have a subtle `bg-merkle-navy/10` background.

### 2.3 Real-time Kanban Board (US-024)
*   **Visual Feedback for WebSocket Events:**
    *   When a card is moved by *another* user, it should animate smoothly to its new column.
    *   Upon landing, the card should briefly glow (`ring-2 ring-merkle-navy transition-all duration-1000`) before fading back to its normal state, indicating an external change.
