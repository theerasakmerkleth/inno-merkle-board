## 1. Architectural Vision: "The Pristine Enterprise OS"

To solve the cognitive overload commonly found in legacy tools like Jira, Version 12.0 completely pivots from the heavy dark/charcoal themes to a **Minimalist "Clean White" Baseline (Light Mode First)**. This approach uses ample negative space (whitespace) as the primary structuring element, removing harsh borders and heavy background blocks. 

The Merkle Brand Guidelines are applied strategically: the stark white canvas allows the `merkle-red` primary actions and `merkle-navy` data visualisations to guide the user's eye naturally without causing visual fatigue.

**Core UX Principles for v12.0:**
1.  **Whitespace as Structure:** Group high-density data (Custom Fields, Sub-tasks, Activity Logs) using spacing and very subtle typography scale changes, rather than borders or card backgrounds.
2.  **Brand-Accented Minimalism:** The background (`--background`) is pristine white (`#FFFFFF`). Borders (`--border`) are an ultra-light gray (`zinc-200` or `#E4E4E7`). `merkle-red` is reserved strictly for primary buttons (`<Button>`), active states, and critical alerts.
3.  **Zero-Context-Switching:** Maintain the `<Sheet>` slide-over for Task Details and the `<Command>` palette for global navigation, ensuring they feel like floating, lightweight layers over the crisp white board.
4.  **A11y & High Contrast:** Ensure WCAG 2.1 AA compliance. Text is primarily a deep, readable charcoal/slate (`zinc-900` or `#18181B`), avoiding pure black for softer readability.

## 2. Design System Customization (Tailwind & shadcn/ui)

### 2.1 Core Token Mapping (Merkle Brand)
*   **Base/Canvas:** `bg-white` (`#FFFFFF`).
*   **Surface/Cards:** `bg-white` with a very subtle, soft shadow (`shadow-sm`, `shadow-black/5`) to lift them off the canvas slightly.
*   **Borders & Dividers:** `border-zinc-200` (`#E4E4E7`). Extremely subtle.
*   **Typography:** 
    *   Headings (`h1-h4`): `text-zinc-900` (`#18181B`), font-semibold, tight tracking.
    *   Body/Muted Text: `text-zinc-500` (`#71717A`) for secondary information (dates, small labels).
*   **Primary Brand Color (Merkle Red):** `bg-merkle-red` (`#e4002b`). Used for primary CTAs (e.g., "Create Issue", "Save Task") and active navigational borders.
*   **Secondary Brand Color (Merkle Navy):** `text-merkle-navy` (`#002C5F`). Used for data visualisation, progress bars, and AI Agent highlights (e.g., `bg-merkle-navy/5` for AI comment backgrounds).

### 2.2 Global Layout & App Shell
*   **Left Sidebar:** 
    *   Changes from dark charcoal to a crisp white (`bg-white`) or off-white (`bg-zinc-50/50`) background, separated from the main content by a delicate `border-r border-zinc-200`.
    *   Active menu items have a subtle `bg-merkle-red/5` background and `text-merkle-red` typography.
*   **Top App Bar:** 
    *   `bg-white` with a translucent blur (`backdrop-blur-md bg-white/80`) and a `border-b border-zinc-200`.
    *   The `Cmd + K` search bar is styled as a soft gray pill (`bg-zinc-100 hover:bg-zinc-200/80 text-zinc-500`).

## 3. Core Views UI Specification

### 3.1 The Kanban Board (Agile Execution)
*   **Background:** The board canvas is an ultra-light gray (`bg-zinc-50`) to allow the white task cards to pop.
*   **Column Headers:** Sticky `bg-zinc-50`, text is `zinc-500`, uppercase, tracking-wider, `text-xs`.
*   **Task Cards:** 
    *   `bg-white` with a delicate `border border-zinc-200/60` and `shadow-sm`.
    *   Hover state: `hover:shadow-md hover:border-zinc-300 transition-all duration-200`.
    *   Priority lines: A thin left-border (`border-l-2`) using brand/semantic colors (e.g., `border-merkle-red` for High).
*   **Drag & Drop:** While dragging, the card lifts (`scale-105 shadow-xl`) and the dropzone shows a dashed outline `border-2 border-dashed border-zinc-300 bg-zinc-100/50`.

### 3.2 The Issue View (Slide-over `<Sheet>`)
*   **Backdrop:** `bg-zinc-900/20 backdrop-blur-sm` (much lighter than the previous black/60).
*   **Sheet Body:** Pure `bg-white`, `border-l border-zinc-200`, `shadow-2xl`.
*   **Rich Text Editor (TipTap):** The editing area has no heavy border, just a subtle hover state (`hover:bg-zinc-50 rounded-md`). The toolbar uses minimalist icons (`text-zinc-500`).
*   **Sub-tasks (Checklists):** Checkboxes (`shadcn/Checkbox`) use `merkle-navy` when checked. The progress bar is a thin `h-1` `bg-zinc-100` track with a `bg-merkle-navy` fill.
*   **Comments:** 
    *   Human comments: `bg-white`, no border, separated by generous whitespace and a tiny avatar.
    *   AI Agent comments: `bg-merkle-navy/5`, `border border-merkle-navy/10`, `rounded-lg` to subtly distinguish them.

### 3.3 My Tasks (Split-Pane Inbox)
*   **Left Pane (Task List):** `bg-zinc-50/50 border-r border-zinc-200`. Selected task uses `bg-white shadow-sm border border-zinc-200 z-10 scale-[1.02]`.
*   **Right Pane (Details):** Pure `bg-white`. The reading experience is similar to a pristine document (Medium/Notion-like typography for the description).

### 3.4 Enterprise Analytics (Bento-Box)
*   **Cards:** `bg-white border border-zinc-200 rounded-xl shadow-sm`.
*   **Charts/Bars:** Using `merkle-red` and `merkle-navy` exclusively.
*   **Typography:** Large metric numbers in `text-zinc-900 font-semibold tracking-tight`.

## 4. Interaction States & Edge Cases

*   **Empty States:** Display a sleek, single-color line-art icon (e.g., an empty inbox) in `zinc-300` with a subtle `merkle-red` accent. Text is `text-zinc-500`.
*   **Loading States:** Use `<Skeleton className="bg-zinc-100" />` to create a soft, pulsing effect that blends seamlessly into the white background. Avoid harsh gray skeleton blocks.
*   **Optimistic UI:** State changes (like moving a card or checking a sub-task) update the UI immediately with a snappy `duration-150` transition, fading a small `text-green-600` checkmark momentarily on success.

## 5. Pixel-Perfect Developer Handoff

### Developer Component Checklist (`shadcn/ui`):
Run these commands to prepare the base components:
1.  `npx shadcn-ui@latest add sheet` (For Task Modal)
2.  `npx shadcn-ui@latest add dialog` (Base for Modals)
3.  `npx shadcn-ui@latest add command` (For `Cmd+K` Palette)
4.  `npx shadcn-ui@latest add checkbox` (For Sub-tasks)
5.  `npx shadcn-ui@latest add skeleton` (For Loading states)
6.  `npx shadcn-ui@latest add dropdown-menu` (For Notification Bell & Profile)

### Tailwind Overrides (`tailwind.config.js` Action Required):
The Developer MUST update the theme colors to match the "Clean White" aesthetic:
*   Remove `bg-merkle-charcoal` from root `<body>` and replace with `bg-white` or `bg-zinc-50`.
*   Replace all `text-zinc-100` (dark mode text) with `text-zinc-900`.
*   Replace all `border-zinc-800` with `border-zinc-200`.
*   Replace `bg-zinc-900` card backgrounds with `bg-white shadow-sm border border-zinc-200`.
