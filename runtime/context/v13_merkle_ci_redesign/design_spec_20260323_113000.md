## 1. Design System & UX Architecture

### Token Mapping (Merkle Brand Guidelines & Clean White Baseline)
To align with the Merkle Brand Identity and the "Clean White" aesthetic, we will override `shadcn/ui` variables in `tailwind.config.js`:
-   `--background`: `#FFFFFF` (Stark White for maximum readability and negative space).
-   `--foreground`: `#1C1C1E` (Dark Charcoal, high contrast against white).
-   `--card`: `#FFFFFF` (Cards float on the white canvas).
-   `--card-foreground`: `#1C1C1E`.
-   `--popover`: `#FFFFFF`.
-   `--popover-foreground`: `#1C1C1E`.
-   `--primary`: `#000000` (Merkle Black for primary actions).
-   `--primary-foreground`: `#FFFFFF`.
-   `--secondary`: `#F4F4F5` (Very subtle light gray for secondary actions).
-   `--secondary-foreground`: `#1C1C1E`.
-   `--muted`: `#F4F4F5`.
-   `--muted-foreground`: `#71717A` (Gray for placeholder text and less important metadata).
-   `--accent`: `#E4E4E7` (Subtle gray for hover states).
-   `--accent-foreground`: `#1C1C1E`.
-   `--destructive`: `#DC2626` (Standard Red for errors).
-   `--destructive-foreground`: `#FAFAFA`.
-   `--border`: `#E4E4E7` (Extremely light, barely-there borders to create structure without visual weight).
-   `--input`: `#E4E4E7`.
-   `--ring`: `#000000` (High contrast focus ring for Accessibility).
-   `--radius`: `0.5rem` (Slightly rounded corners for a modern feel).

### Core Views UI Specification
1.  **Global Shell (Ample Whitespace):** The Topbar and Sidebar will use `#FFFFFF` backgrounds with a subtle bottom/right border (`border-b border-border`). This removes the "boxed-in" feel, relying on spacing to separate navigation from content.
2.  **The Board (Kanban):**
    -   *Columns:* No background color. Only a subtle column header (`font-semibold`).
    -   *Cards:* White background with a very delicate shadow (`shadow-sm`) and an almost invisible border (`border border-slate-100`). This ensures cards pop slightly off the stark white canvas. Drag-and-drop dropzones will highlight with a subtle `--accent` background.
3.  **The Issue View (Slide-over `<Sheet>`):**
    -   Triggers from the right side. The sheet itself is stark white (`bg-background`).
    -   *Layout:* Left 2/3 for Title, Rich Text Description, Checklists, and Activity. Right 1/3 for Metadata (Assignee, Status, Priority, Attachments).
    -   *Typography:* High hierarchy. Large, bold headers (`text-2xl font-bold`) and standard legible body text (`text-sm`).

## 2. Interaction States & Edge Cases

*   **Empty States:** Utilize centered, minimalist Material Icons (e.g., `<Icon name="inbox" size="xl" className="text-muted-foreground/50" />`) with short, actionable text.
*   **Loading States:** Use `shadcn/ui` `<Skeleton>` components that pulse softly in light gray (`bg-muted`), maintaining the shape of the expected content.
*   **Error States:** Inline `<Alert variant="destructive">` for form errors. Toast notifications for failed actions (e.g., failed to move a card).
*   **Optimistic UI:** When dragging a Kanban card, the UI instantly reflects the new position. If the server request fails, the card snaps back, and a Toast error is displayed.

## 3. Pixel-Perfect Developer Handoff

### `shadcn/ui` Components Required
The following components must be installed via `npx shadcn@latest add [component]`:
-   `sheet`
-   `skeleton`
-   `toast`
-   `alert`
-   `button`
-   `input`
-   `command`
-   `dropdown-menu`

### API & Interaction Requirements
-   **Task Details:** The UI expects a structured JSON object containing Title, Description (HTML/Markdown string), Checklist Array, and Attachments Array.
-   **Real-time Kanban (US-024):** The frontend needs to listen to a WebSocket channel (e.g., `project.{id}.board`) for `TaskMoved` events. Upon receiving the event, the frontend state must update the card position without a full page reload.

### Accessibility (A11y) Notes
-   All interactive elements must have visible focus rings (`focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`).
-   The Kanban board drag-and-drop must be keyboard accessible (e.g., using Space to pick up, Arrows to move, Space to drop).
-   Ensure contrast ratios for all text against the `#FFFFFF` background meet WCAG 2.1 AA standards.
