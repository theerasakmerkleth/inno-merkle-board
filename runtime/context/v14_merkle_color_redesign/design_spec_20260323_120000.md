## 1. Design System & UX Architecture

### Token Mapping (Merkle Core Colors & Grayscale)
Based on the provided `collor application.webp` palette, we will infuse the application with Merkle's official blues, reds, and grays, while maintaining a clean, enterprise-ready look. We will map these colors directly into our `shadcn/ui` and Tailwind configuration to build a cohesive visual hierarchy.

**Extracted Color Palette:**
*   **Blues:** Sky Blue (`#0391F2`), Cobalt (`#0328D1`), Klein (`#031FA4`), Merkle Blue (`#040E4B`).
*   **Reds:** Merkle Red (`#DD3039`), Claret (`#711A2B`).
*   **Grayscales:** Mid Gray Tint (`#D6D6DF`), Mid Gray (`#AEAEBC`), Dark Gray Tint (`#8888A1`), Dark Gray (`#60607D`).

### Core Views UI Specification
1.  **Global Shell & Typography:**
    -   The application will use the very dark **Merkle Blue (`#040E4B`)** as the primary text color (`foreground`), giving it a premium, branded feel.
    -   Borders (`border`) will use the soft **Mid Gray Tint (`#D6D6DF`)**, reducing visual clutter while separating content.
2.  **The Kanban Board:**
    -   *Cards:* White background floating on the canvas, using subtle gray borders (`border-border`).
    -   *Tags/Priorities:* Use **Merkle Red (`#DD3039`)** for High priority, **Sky Blue (`#0391F2`)** for Medium, and **Dark Gray (`#60607D`)** for Low.
3.  **Interactive Elements (Buttons, Inputs):**
    -   Primary buttons will use **Cobalt (`#0328D1`)** for maximum contrast and brand identity.
    -   Focus states (`ring`) will utilize the vibrant **Sky Blue (`#0391F2`)** to ensure accessibility and keyboard navigation visibility.
    -   Hover states will softly transition using the **Mid Gray (`#AEAEBC`)**.

## 2. Accessibility (A11y) & Contrast Ratios
-   **Merkle Blue (`#040E4B`)** on White (`#FFFFFF`) provides exceptional contrast (AAA rating) for all body text.
-   **Cobalt (`#0328D1`)** with White text (`#FFFFFF`) passes AAA rating for interactive buttons.
-   **Sky Blue (`#0391F2`)** focus rings provide stark visibility against white and gray backgrounds, satisfying WCAG 2.1 Focus Appearance criteria.
-   Avoid using `Mid Gray Tint` for text; it should strictly be used for borders and disabled/muted backgrounds.

## 3. Pixel-Perfect Developer Handoff
### CSS Variables Update Action
The Developer must update `resources/css/app.css` (both `:root` and `.dark` variants) to map exactly to the hex codes provided in Section 1.

### Dark Mode Adjustments (Guidance)
If dark mode is supported, flip the primary canvas to a very dark shade (e.g., `#0A0B10`), use `Sky Blue` for primary actions instead of Cobalt (to prevent bleeding), and use `Mid Gray` for text to prevent eye strain.
