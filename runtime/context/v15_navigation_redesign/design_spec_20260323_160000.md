# UX/UI Design Specification: Navigation & User Journey Redesign
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 15.0 (Global Navigation & Creation Flow Redesign)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Empathy Mapping
**User Pain Point:** *"ไปหน้าสร้าง project หรือ board ไม่ถูก งงมาก" (I don't know how to navigate to create a project or a board. It's very confusing.)*
- **Current State:** Projects are only listed as links in the Sidebar and Command Palette. The top bar only has a "Create Issue" button (which is unimplemented). There is no "Projects Hub" to view or create projects, and creating a new board within a project is hidden or non-existent in the UI.
- **Goal:** Redesign the Information Architecture (IA) and User Journey so that creating Projects, Boards, and Tasks is intuitive, centralized, and adheres to the "Clean White" & Merkle CI aesthetic without adding modal clutter.

## 2. Information Architecture (IA) Redesign

### 2.1 The Global "Create" Action (Topbar)
Instead of a single "Create Issue" button, we will implement a **Global Create Menu** (`<DropdownMenu>`) in the Top App Bar. 
- **Trigger:** A prominent Cobalt (`#0328D1`) button labeled **"Create..."** with a `+` icon.
- **Options:**
  - `New Task` (Shortcut: `C`)
  - `New Project` (Restricted to Admins/PMs)
  - `New Board` (Contextual, active if currently viewing a project)

### 2.2 The Workspace Hub (Projects List)
We need a dedicated page at `/projects` that serves as the "Home" for all workspace projects.
- **Layout:** A clean grid layout (`grid-cols-1 md:grid-cols-3`) displaying Project Cards.
- **Project Card:** Displays Project Name, Key, Status, and Active Member count. Floating on the white canvas with a subtle `border-border`.
- **Empty State:** If no projects exist, show a centered illustration with a "Create Your First Project" CTA.

### 2.3 The Sidebar Enhancements
- **Projects Section Header:** Add a small, subtle `+` icon next to the "Projects" label in the sidebar. Clicking this immediately opens the "Create Project" flow.
- **Active State:** The currently active project must have a distinct visual highlight (e.g., `bg-accent` or a left-border using Cobalt `#0328D1`) so the user knows where they are.

### 2.4 The Board Navigation (Within a Project)
When viewing a project (`/projects/{key}/boards`):
- Next to the list of existing Boards (Sprint 1, Backlog, etc.), there must be an explicit **"+ New Board"** tab or button.
- We will use an inline creation flow: Clicking "+ New Board" changes the tab into a small `<Input>` field to type the board name, followed by an `Enter` keypress to save. This avoids a heavy modal.

## 3. UI Flow & State Management

**Flow 1: Creating a Project**
1. User clicks "Create..." in Topbar -> Selects "New Project" OR clicks the `+` in the Sidebar.
2. A slide-over panel (`<Sheet>`) opens from the right (to keep context).
3. Form fields: Project Name, Key (auto-generated but editable), Status.
4. On Submit: Optimistic UI immediately adds the project to the sidebar and navigates the user to the newly created empty board.

**Flow 2: Creating a Board**
1. User is on the Kanban Board view (`/projects/CORE/boards`).
2. User clicks the "+ New Board" tab next to the existing board names.
3. An inline input appears. User types "Sprint 2" and hits Enter.
4. The board is created via a `POST` request, and the view switches to the empty "Sprint 2" board.
