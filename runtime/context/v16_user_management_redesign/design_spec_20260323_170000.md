# UX/UI Design Specification: User Management Redesign
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 16.0 (User Management Redesign)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Empathy Mapping
**User Pain Point:** The "Directory" page feels disconnected from the rest of the application. It uses a standalone layout with a custom header, breaking the navigation paradigm established by the `AppLayout`. Furthermore, the term "Directory" sounds like a passive phonebook rather than an active administrative interface.
- **Current State:** A full-page layout without the global sidebar or top app bar.
- **Goal:** Redesign the feature into an integrated "User Management" hub that utilizes the standard `AppLayout` wrapper, aligns with the "Clean White" Merkle CI aesthetic, and provides a cohesive Enterprise experience for Admins.

## 2. Information Architecture (IA) Redesign

### 2.1 Terminology & Navigation Update
- **Rename:** "Directory" -> "User Management".
- **Sidebar Icon:** Change from `group` to `manage_accounts` to signify active administration.
- **Header/Breadcrumbs:** Align with standard pages (e.g., `AppLayout breadcrumbs={<span>User Management</span>}`).

### 2.2 The "User Management" View (Index.tsx)
- **Layout Container:** Wrapped inside the standard `flex-1 overflow-y-auto p-4 md:p-8 bg-background` structure to match Analytics and Projects.
- **Page Header:** 
  - **Title:** "User Management" (`text-2xl font-semibold text-foreground`).
  - **Subtitle:** "Manage workspace members, roles, and access controls." (`text-sm text-muted-foreground`).
  - **Primary Action:** The "Invite User" button should be elevated to a Primary Button (`bg-primary text-primary-foreground`) using Cobalt (`#0328D1`) to match the Global Create actions.
- **Data Table:**
  - Encapsulate the raw `<table>` inside a Card container (`bg-card border border-border rounded-lg shadow-sm`) so it floats cleanly on the white canvas.
  - Maintain the existing hover states (`hover:bg-accent`), context menu (using `bg-card` and `border-border`), and status indicators (Emerald for Active, Muted for Disabled).

## 3. UI Flow & State Management
- **Role Changes:** The "Change Role" dropdown options remain inside the three-dots context menu to reduce visual clutter in the table rows.
- **Access Revocation:** Destructive actions (Disable Access) remain styled with Merkle Red (`text-destructive`).
