# UX/UI Design Specification: Project Management Board & Task View Redesign
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 17.0 (Board & Task Slide-over Redesign)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Empathy Mapping
**User Pain Point:** The Kanban Board and Issue View feel clunky. The Task modal overlays the whole screen, breaking the user's flow and context of the Kanban board.
- **Goal:** Redesign the Kanban Board and Task Issue View to mirror modern Enterprise Agile tools (like Linear or Jira). Use a Slide-over (`<Sheet>`) pattern for tasks to maintain board context, increase data density without clutter, and apply the Merkle Brand CI (Clean White).

## 2. Information Architecture (IA) Redesign

### 2.1 The Kanban Board (Data Grid & Columns)
- **Column Headers:** Simple, stark headers with a subtle badge showing the count of tasks. No heavy colored background boxes.
- **The Board Background:** Light muted background (`bg-muted/10` or `bg-background`) with columns separated by negative space, allowing the stark white Task Cards (`bg-card`) to pop out.
- **Card Aesthetics:**
  - Crisp `border-border` with a subtle drop shadow (`shadow-sm`).
  - Left border indicates priority (High = Merkle Red, Medium = Sky Blue, Low = Dark Gray).
  - Use `Avatar` circles for assignees and clear typography for Task Keys (e.g., `[CORE-12]`).

### 2.2 The Issue View (Slide-over / `<Sheet>`)
- Replace the center-screen Modal with a right-aligned Slide-over panel. This ensures the user can still reference the board while reading/editing the task.
- **Width:** 50% or 600px width on desktop to accommodate high-density data.
- **Layout Structure:**
  - **Header:** Task Key (`[CORE-12]`), Title (inline editable), Status dropdown, and Close button.
  - **Left/Main Pane:** Rich Text Description (TipTap editor), Checklists, and Activity/Comments stream.
  - **Right/Sidebar Pane (Metadata):** Assignee, Priority, Board, Labels/Tags, and Attachments.
- **Typography & Spacing:** Use ample whitespace (negative space) between sections (Description, Checklists, Comments). Utilize `Merkle Blue` for primary headings and `Mid Gray` for metadata labels.

## 3. UI Flow & State Management
- **Task Interaction:** Clicking a card slides open the details pane smoothly from the right.
- **Inline Editing:** Priority, Status, and Assignee should be `<select>` or custom dropdown components styled minimally so they don't look like heavy form fields unless focused.
- **Comments Flow:** The comment input box remains pinned at the bottom of the activity stream or neatly below the description, using the `Sky Blue` focus ring when active.

## 4. Developer Handoff (Post-Approval)
