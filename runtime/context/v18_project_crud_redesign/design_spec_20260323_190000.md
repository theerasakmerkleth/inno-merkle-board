# UX/UI Design Specification: Project CRUD Redesign
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 18.0 (Project CRUD Accessibility Redesign)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Empathy Mapping
**User Pain Point:** *"UX/UI please redesign feature CRUD project now to hard for me"* 
- **Current State:** While users can *Create* projects via the Workspace Hub, and *Read* them via the Dashboard, they cannot naturally *Update* or *Delete* them. There are zero navigational links to the `/projects/{key}/settings` page in the entire UI. This makes managing (CRUD) a project extremely frustrating and technically inaccessible to non-developers.
- **Goal:** Expose the Update and Delete (Settings) flows natively within the UI without overwhelming the clean interface. Bring the Project CRUD lifecycle into plain sight for Managers/Admins, adhering strictly to the Merkle Brand CI.

## 2. Information Architecture (IA) Redesign

### 2.1 Workspace Hub (`/projects`) - The Entry Point
- **Enhancement:** On each Project Card, add a contextual "Settings" (Gear icon) button in the top-right corner. This button will only be visible/clickable if the user has Manager or Admin privileges.
- **Aesthetic:** Use a subtle `text-muted-foreground hover:text-foreground hover:bg-muted` circular button that doesn't distract from the main card title.

### 2.2 The Kanban Board (`/projects/{key}/boards`) - Contextual Management
- **Enhancement:** In the Top Header of the Kanban Board, next to the "Create Task" button, add a secondary "Project Settings" button for authorized users.
- **Aesthetic:** Use a standard outline or ghost button (`bg-transparent border border-border text-foreground hover:bg-accent`) with a Settings gear icon to differentiate it from the primary Cobalt action buttons.

### 2.3 Creation Flow Refinements
- **Enhancement:** The Slide-over panel for "Create Project" is mostly fine, but we need to ensure the auto-generation of the "Project Key" feels magical and effortless. The helper text will be refined to guide the user naturally.

## 3. UI Flow & State Management
1. **Navigating to Settings:** An Admin clicks the gear icon on the Workspace Hub -> Routes to `/projects/{key}/settings`.
2. **Navigating from Kanban:** A Project Manager clicks "Project Settings" in the header -> Routes to `/projects/{key}/settings`.
3. **Delete/Update Flow:** Once inside Settings, the user can easily rename the project or utilize the Merkle Red "Danger Zone" to delete the project (which cascades to boards and tasks).
