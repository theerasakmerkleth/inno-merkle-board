# UX/UI Design Specification: Navigation Menu (Sidebar) Redesign
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 19.0 (Menu Legibility & Active States)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Empathy Mapping
**User Pain Point:** *"UX/UI redesign menu now i thik hard to use for me"*
- **Current State:** The main Sidebar menu relies on hardcoded, low-contrast text colors (`text-zinc-400` on a white background). Furthermore, there is absolutely no visual indication of the "Active State" (which page the user is currently viewing). This leads to a severe lack of orientation and wayfinding, confusing users.
- **Goal:** Redesign the Sidebar to be highly legible, fully accessible (A11y AA contrast), and explicitly clear about the user's current location in the app, while maintaining the Clean White & Cobalt (Merkle CI) theme.

## 2. Information Architecture (IA) Redesign

### 2.1 Contrast & Legibility
- **Text Colors:** All inactive menu items will transition from the illegible `text-zinc-400` to the semantic `text-muted-foreground` (which we defined as Dark Gray `#60607D` in the CI). This guarantees WCAG text contrast compliance on the white canvas.
- **Icons:** Material Icons will match the text color exactly, maintaining a balanced visual weight.

### 2.2 The "Active State" Paradigm
- To solve the "Where am I?" problem, the sidebar must evaluate the current URL.
- **Active Styling:** When a user is on a specific page (e.g., My Tasks), that menu item will be styled with `bg-primary/10` and `text-primary` (Cobalt Blue), while the font weight increases to `font-medium`. 
- **Inactive Styling:** Hovering over inactive items will trigger a soft `hover:bg-muted` state rather than immediately jumping to the primary color, keeping the interface calm.

### 2.3 Structural Cleanup
- **Borders:** Purge legacy `border-zinc-200` classes and replace them with the standard `border-border`.
- **Project List:** Ensure the project links also evaluate their active state. If the user is currently viewing `[CORE]`, that project link in the sidebar should be highlighted.

## 3. UI Flow & State Management
- **Inertia `usePage().url`:** The React component will read the current URL path to dynamically inject active Tailwind CSS classes into the `<Link>` components.

## 4. PO Validation Loop (Pending)
*This update directly addresses the accessibility and wayfinding confusion reported by users. It requires zero backend changes and strictly utilizes existing Merkle CI tokens.*
