## 1. System Architecture (GCP + Laravel)

-   **Frontend:** React Single Page Application (SPA) utilizing Inertia.js within the Laravel monolith. `shadcn/ui` used for presentation.
-   **Backend:** Laravel 13 application deployed as a containerized service on **Google Cloud Run** for serverless scaling.
-   **Database:** **Google Cloud SQL (PostgreSQL 15)** handling all relational data, including user mapping, Kanban state, and webhook histories.
-   **Cache & Queue:** **Google Cloud Memorystore (Redis)** configured for session persistence, rapid Kanban board state caching, and processing asynchronous background jobs (like parsing Git Webhooks).
-   **Security Boundaries:**
    -   *Web Users (PMs, Developers):* Protected via standard Laravel Session & Cookie-based authentication.
    -   *AI Agents (Digital Workers):* Protected via stateless API tokens issued by **Laravel Sanctum**.
    -   *External Services (Git Webhooks):* Secured via payload signature validation (e.g., `X-Hub-Signature`).

## 2. Database Schema Design (PostgreSQL)

### 2.1 Core Entities

**`users` Table**
- `id` (PK)
- `name` (String)
- `email` (String, Unique)
- `role` (Enum: 'admin', 'manager', 'developer', 'agent')
- `git_username` (String, Nullable) - Used to map webhook commits to specific developers.

**`projects` Table**
- `id` (PK)
- `name` (String)
- `status` (Enum: 'active', 'archived')
- `timestamps`

**`tasks` Table**
- `id` (PK)
- `project_id` (FK -> projects.id)
- `title` (String)
- `description` (Text, Nullable)
- `status` (Enum: 'todo', 'in_progress', 'awaiting_review', 'done')
- `priority` (Enum: 'low', 'medium', 'high')
- `assignee_id` (FK -> users.id, Nullable) - Can be a human or an AI agent.
- `requires_human_review` (Boolean, Default: true) - The "Human-in-the-loop" toggle.
- `timestamps`

### 2.2 Integration Entities

**`git_webhook_events` Table**
- `id` (PK)
- `task_id` (FK -> tasks.id)
- `provider` (String) - e.g., 'github', 'gitlab'
- `commit_hash` (String)
- `author_git_username` (String)
- `diff_url` (String)
- `timestamps`

**`ai_agent_submissions` Table**
- `id` (PK)
- `task_id` (FK -> tasks.id)
- `agent_id` (FK -> users.id)
- `payload_data` (JSONB) - Structured output from the AI agent.
- `review_status` (Enum: 'pending', 'approved', 'rejected')
- `reviewed_by` (FK -> users.id, Nullable)
- `timestamps`

## 3. API Contracts (RESTful)

### 3.1 AI Agent API (Protected by Sanctum Middleware)
-   **Endpoint:** `GET /api/v1/agent/tasks`
    -   *Purpose:* Retrieves a list of tasks where the current authenticated Agent is the `assignee_id` and status is not 'done'.
    -   *Response:* 200 OK, JSON array of Task objects.
-   **Endpoint:** `POST /api/v1/agent/tasks/{id}/submit`
    -   *Purpose:* Allows the Agent to submit completed work.
    -   *Payload:* `{ "result_payload": { ... } }`
    -   *Action:* Creates an entry in `ai_agent_submissions`, updates task status to `awaiting_review`, and triggers a notification.
    -   *Response:* 201 Created.

### 3.2 Webhook Intake (Protected by Signature Middleware)
-   **Endpoint:** `POST /api/v1/webhooks/git`
    -   *Purpose:* Receives push/PR events from GitHub/GitLab.
    -   *Headers:* Requires valid `X-Hub-Signature-256`.
    -   *Action:* Dispatches a Redis queued job to parse the payload, identify `#TASK-ID`, log the commit, and advance the task status.
    -   *Response:* 202 Accepted (Processing deferred to queue).

### 3.3 Dashboard API (Protected by Session Auth)
-   **Endpoint:** `GET /api/v1/dashboard/metrics`
    -   *Purpose:* Serves aggregated data for the Merkle-style PM dashboard.
    -   *Response:* JSON containing `resource_loading` (tasks per user) and `project_health` (completion percentages).

## 4. Handover to Developer
1.  **Repository Setup:** Initialize the Laravel project with Inertia.js (React) in the `./output/code/` directory.
2.  **Dependencies:** Ensure `laravel/sanctum` is installed for the API layer.
3.  **Database:** Create migrations based on the schema outlined above, paying special attention to the `JSONB` column for AI submissions.
4.  **UI Implementation:** Follow the design guidelines (Slate/Zinc, Deep Red accents) while building the React components.
