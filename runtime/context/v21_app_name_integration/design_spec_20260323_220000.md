# UX/UI Design Specification: Dynamic App Name Integration
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 21.0 (Dynamic Branding & White-labeling Readiness)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Empathy Mapping
**User Pain Point:** The application name is currently hardcoded as "TaskFlow AI" or simply just displaying the Merkle logo without text. If the business decides to rebrand the platform or deploy it for a specific client (white-labeling), developers would have to hunt down hardcoded text strings across the React components.
- **Goal:** Redesign the UI headers to dynamically render the `APP_NAME` environment variable. This ensures a single source of truth for the platform's identity, aligning with Enterprise architecture standards.

## 2. Information Architecture (IA) Redesign

### 2.1 The Login Page (`Pages/Auth/Login.tsx`)
- **Current State:** Hardcoded text "Enterprise Task Management Platform" under the logo.
- **Enhancement:** Display the dynamic `app_name` alongside or directly below the Merkle logo to reinforce the product's identity before the user logs in.
- **Aesthetic:** `text-lg font-semibold tracking-tight text-foreground` to match the Clean White Merkle CI.

### 2.2 The Global Sidebar (`Layouts/AppLayout.tsx`)
- **Current State:** Only the Merkle Logo is shown at the top of the sidebar.
- **Enhancement:** Add the `app_name` text next to the logo. This provides constant brand reinforcement without taking up too much vertical space.
- **Aesthetic:** `font-bold text-foreground text-sm tracking-tight`.

### 2.3 The HTML Document Title (`resources/views/app.blade.php`)
- **Enhancement:** Update the `<title>` tag and use Inertia's `<Head>` component throughout the app to dynamically reflect the `app_name` and the current page context (e.g., `My Tasks - Merkle TaskFlow`).

## 3. UI Flow & State Management
- **Inertia Global Props:** The backend will inject `appName` into the global Inertia shared data via `HandleInertiaRequests.php`.
- **Frontend Usage:** React components will extract `appName` using `usePage().props.appName` to render the text.

## 5. Developer Handoff (Post-Approval)
- Update `HandleInertiaRequests.php` to share `config('app.name')` as `appName`.
- Update `Login.tsx` and `AppLayout.tsx` to read and display `appName`.
- Run `npm run build`.
