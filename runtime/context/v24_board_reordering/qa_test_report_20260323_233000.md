# QA Test Report: Kanban DnD Integrity
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 24.1 (DnD Regression Testing)
**Date:** 2026-03-23
**QA Engineer:** Senior QA Automation Engineer & SDET
**Status:** 🔴 NO-GO (Functional Regressions)

## 1. Executive Summary
The implementation of "Nested Drag and Drop" (v24.0) has introduced functional regressions in the core Kanban workflow. While column reordering works, task movement between different lanes is unreliable and frequently fails or results in data corruption (incorrect ranking).

## 2. Defect Log

### 🔴 BUG-DnD-001: Cross-Lane Move Failure
- **Severity:** CRITICAL
- **Description:** Dragging a task card from one column to another empty column (or empty space below tasks) fails to persist or trigger the UI update.
- **Root Cause Suspect:** The `handleDragEnd` logic relies on `over.id` to find the target column. When dropping on a column's empty area, the collision detection `closestCenter` may prioritize the parent `SortableContext` (Column level) incorrectly or fail to map the column ID back to the `targetColumnId` variable.

### 🟠 BUG-DnD-002: Index Corruption on Drop (-1 Index)
- **Severity:** MAJOR
- **Description:** Dropping a task within its own lane but on the column header/empty space causes the task to jump to the very bottom, even if dragged to the top.
- **Technical Detail:** `sourceTasks.findIndex(t => t.id.toString() === targetId)` returns `-1` because `targetId` is the Column ID, not a Task ID. `arrayMove` with `-1` causes the corruption.

### 🟠 BUG-DnD-003: Missing Type Guard for Tasks
- **Severity:** MAJOR
- **Description:** The task reordering logic in `handleDragEnd` lacks a `type === 'Task'` guard. This causes potential conflicts when dragging other sortable elements (like Columns) near task containers.

## 3. Recommended Fixes for Developer
1. **Implement `type` check:** Wrap task logic in `if (active.data.current?.type === 'Task')`.
2. **Improve Target Logic:** Use the `data.current` property from the `over` object to determine if the target is a `Column` or another `Task` directly, rather than looping and searching.
3. **Switch Collision Algorithm:** Consider `pointerWithin` or `rectIntersection` for better nested container detection.

## 4. The Gatekeeper Decision
**Decision: 🔴 NO-GO**
The Kanban board is the primary work engine. Failure to move tasks accurately is a "Showstopper" for Agile teams. 

**Required Action:** Developer must refactor `handleDragEnd` to explicitly handle `over` targets that are Containers (Columns) vs. Sortable Items (Tasks).
