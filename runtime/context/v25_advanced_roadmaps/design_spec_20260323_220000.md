# UX/UI Design Specification: Advanced Roadmaps (Timeline & Dependencies)
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 25.0 (Enterprise Strategic Planning)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Strategic Vision
**Enterprise Context:** As projects grow, simple Kanban boards are insufficient for high-level planning. Stakeholders need to visualize task durations, overlap, and critical path dependencies (blockers) across a linear timeline.
- **Goal:** Design a "Strategic Roadmap" view that provides a Gantt-style timeline while maintaining the Merkle Brand's "Clean White" aesthetic. Focus on high data density, intuitive drag-to-reschedule interactions, and clear dependency visualization.

## 2. Information Architecture (IA) Redesign

### 2.1 The Roadmap Hub (`/projects/{key}/roadmap`)
- **Integration:** Accessible via a new "Roadmap" tab in the project header, alongside "Board" and "Settings".
- **Layout:** Two-pane horizontal split.
    - **Left Pane (Task List):** Sticky sidebar showing Task Key and Title.
    - **Right Pane (Timeline Grid):** A scrollable canvas displaying Date headers (Month/Week/Day) and Task Bars.

### 2.2 Visual Components (Merkle CI Themed)
- **Task Bars:** 
    - **Background:** Primary Cobalt (`#0328D1`) for active tasks, Muted Gray (`#AEAEBC`) for completed tasks.
    - **Corners:** Rounded (`--radius: 0.5rem`) to match the shadcn/ui design language.
    - **Indicators:** Small icons inside the bar showing Assignee Avatar and Priority color dot.
- **Dependency Links (Arrows):**
    - **Aesthetic:** Subtle curved lines connecting the end of one task to the start of another.
    - **Color:** Merkle Red (`#DD3039`) if a dependency is causing a schedule conflict (blocking), otherwise Sky Blue (`#0391F2`).
- **Milestones:** Diamond-shaped markers on the timeline for key project deadlines.

## 3. Interaction & Micro-interactions

### 3.1 Timeline Kinetics
- **Drag-to-Reschedule:** Users can grab the center of a task bar to move it across the timeline (updates `start_date` and `due_date`).
- **Resize-to-Extend:** Users can grab the edges of a task bar to extend or shorten its duration.
- **Dependency Drawing:** A "connector" handle appears on hover at the edge of task bars. Dragging from one handle to another task bar creates a "Blocked By" relationship.

### 3.2 Progressive Disclosure
- **Hover States:** Hovering over a task bar shows a mini-tooltip with the full title, assignee, and specific date range.
- **Click State:** Clicking a task bar opens the standard Task Slide-over (`<Sheet>`) from the right, allowing for immediate detail editing without leaving the roadmap.

## 4. Requirement Traceability (US-ROADMAP)
- **F20:** Gantt Chart Visualization.
- **F21:** Task Dependency Mapping (Finish-to-Start).
- **F22:** Dynamic Timeline Scaling (Year/Month/Quarter).

## 5. Developer Handoff (Post-Approval)
- **New Tables Required:** `task_dependencies` (task_id, blocked_by_id).
- **New Columns Required:** `tasks.start_date`, `tasks.due_date`.
- **Frontend Library:** Suggesting `frappe-gantt` or a custom SVG-based React implementation using `dnd-kit` for timeline logic to maintain look-and-feel consistency.
