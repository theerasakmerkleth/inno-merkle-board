# Infrastructure & Architecture Specification: Advanced Roadmaps
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 25.0 (Strategic Timeline Architecture)
**Date:** 2026-03-23
**Architect:** Enterprise Solution Architect (GCP & Laravel)

## 1. Requirements Analysis
To support the **Advanced Roadmaps** feature, the system needs to handle:
- **Temporal Data:** Precise start and end dates for every task.
- **Relational Dependencies:** Many-to-many relationships representing "Blocker" logic.
- **Gantt Calculation Engine:** Logic to detect schedule conflicts (e.g., if a predecessor ends after a successor starts).

## 2. Data Modeling & Strategy

### 2.1 Database Schema Updates

**Table: `tasks` (Update)**
- `start_date`: `date` (nullable) - The planned start date of the work.
- `due_date`: `date` (nullable) - The planned completion date.
- *Index:* Add index to `(project_id, start_date, due_date)` for fast timeline rendering.

**Table: `task_dependencies` (New)**
- `id`: primary key
- `task_id`: foreign key to `tasks.id` (The task that is blocked).
- `blocked_by_id`: foreign key to `tasks.id` (The predecessor task).
- `dependency_type`: `enum` (default: 'finish_to_start').
- `timestamps`

### 2.2 Conflict Detection Logic
A "Scheduling Conflict" occurs if:
`task.start_date < predecessor.due_date`
The API will return a `has_conflict` boolean in the Task Resource for the frontend to render the "Red" dependency lines.

## 3. API Contracts

### 3.1 Roadmap Data Feed
`GET /api/projects/{project}/roadmap`
- **Response:**
  - `tasks`: Array of TaskResources (including `start_date`, `due_date`, `assignee`).
  - `dependencies`: Array of `{ task_id, blocked_by_id }`.

### 3.2 Dependency Management
- `POST /api/tasks/{task}/dependencies` -> `{ blocked_by_id }`
- `DELETE /api/tasks/{task}/dependencies/{blocked_by}`

## 4. Implementation Guidance for Developer

1. **Migrations:** Create a single migration to add columns to `tasks` and create the `task_dependencies` table.
2. **Models:** 
   - Define `dependencies()` (belongsToMany) on the `Task` model.
   - Add `start_date` and `due_date` to `$fillable`.
3. **Frontend Implementation:**
   - Create `Pages/Projects/Roadmap.tsx`.
   - Integrate a Gantt library or build a custom SVG timeline.
   - Use `useForm` to handle drag-to-reschedule (Patching `start_date` and `due_date`).
