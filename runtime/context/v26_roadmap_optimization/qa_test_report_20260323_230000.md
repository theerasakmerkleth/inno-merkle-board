# QA Test Report: Roadmap UI & Data Integrity
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 26.1 (Roadmap Verification)
**Date:** 2026-03-23
**QA Engineer:** Senior QA Automation Engineer & SDET
**Status:** 🔴 NO-GO (Critical Alignment Issues)

## 1. Executive Summary
During the integration testing of the "Advanced Roadmaps" feature (v26.0), several critical UI/UX and data visualization defects were identified. The most severe issue involves a total lack of scroll synchronization between the task labels and the timeline bars, rendering the roadmap unusable for projects with more than 10 tasks.

## 2. Defect Log

### 🔴 BUG-Roadmap-001: Vertical Scroll Desynchronization
- **Severity:** CRITICAL / BLOCKER
- **Description:** The Work Items sidebar (Left Pane) and the Timeline Canvas (Right Pane) have independent scroll containers. 
- **Steps to Reproduce:**
    1. Open the Roadmap for a project with 15+ tasks.
    2. Scroll down on the timeline grid to see the bottom tasks.
- **Expected Behavior:** Both panes scroll together so Task A's label always aligns with Task A's timeline bar.
- **Actual Behavior:** Only the right pane scrolls. The labels on the left remain static, leading to complete data misalignment.

### 🟠 BUG-Roadmap-002: Date Precision Drift (DST/Timezone Risk)
- **Severity:** MAJOR
- **Description:** The `leftDiff` calculation in `getTaskStyle` uses a hard-coded millisecond divisor (`1000 * 60 * 60 * 24`). This does not account for potential DST shifts or sub-day timestamp residues if the server returns non-zeroed hours.
- **Root Cause:** Use of `Math.floor()` on a raw time difference.
- **Recommended Fix:** Use a date-diff utility or ensure both dates are normalized to midnight UTC before calculating the day index.

### 🟡 BUG-Roadmap-003: UI Overflow at "Months" Zoom Level
- **Severity:** MINOR
- **Description:** At the 8px `cellWidth` (Months view), task titles inside the bars are rendered as vertical slivers or cause CSS layout overflow.
- **Expected Behavior:** Titles should be hidden or moved to a tooltip/external label when the bar width is less than a certain threshold (e.g., < 100px).

## 3. Environment Details
- **Browsers:** Chrome 146, Safari 18 (Tested for scroll behavior).
- **Viewports:** Desktop (1440px).

## 4. The Gatekeeper Decision
**Decision: 🔴 NO-GO**
The vertical misalignment (BUG-001) is a functional blocker. Stakeholders cannot reliably interpret which bar belongs to which task.

**Required Action for Dev Team:**
1. Refactor `Roadmap.tsx` to use a single scroll container for both the sidebar and timeline rows, OR implement a `onScroll` sync hook.
2. Improve date parsing using the provided `formatDate` or a dedicated library to prevent 1-day offsets.
3. Update task bar CSS to handle short widths gracefully (e.g., `overflow: hidden` or conditional rendering).
