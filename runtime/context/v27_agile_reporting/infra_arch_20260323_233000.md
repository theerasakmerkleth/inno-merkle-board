# Infrastructure & Architecture Specification: Agile Reporting
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 27.0 (Performance Analytics Engine)
**Date:** 2026-03-23
**Architect:** Enterprise Solution Architect (GCP & Laravel)

## 1. Requirements Analysis
To support the **Agile Reporting** feature, the system must provide high-performance data aggregation for two primary metrics:
- **Burndown (Execution):** Time-series data showing work remaining over the course of a board's duration.
- **Velocity (Capacity):** Comparative data showing output across different boards within a project.

## 2. Data Aggregation Strategy

### 2.1 Burndown Calculation (On-the-fly)
To avoid the overhead of a daily cron snapshot job during the MVP phase, we will utilize the existing `tasks` and `task_transitions` tables to reconstruct the historical state.
- **Algorithm:** 
    1. Determine the date range of the board (e.g., from first task creation to today, or a fixed 2-week window).
    2. For each date in the range:
        - Work Remaining = (Tasks created before/on this date) - (Tasks moved to a 'Done' column before/on this date).
- **Optimization:** Cache the result of this calculation per board/day using Laravel's `Cache` (Redis) with a 1-hour TTL.

### 2.2 Velocity Calculation
- **Commitment:** Total count of tasks ever associated with a board.
- **Completed:** Count of tasks currently in a column containing the word 'Done'.

## 3. API Contracts

### 3.1 Burndown Endpoint
`GET /api/projects/{project}/boards/{board}/stats/burndown`
- **Response:**
  ```json
  {
    "ideal": [{"day": "2026-03-01", "value": 10}, {"day": "2026-03-14", "value": 0}],
    "actual": [{"day": "2026-03-01", "value": 10}, {"day": "2026-03-02", "value": 9}, ...]
  }
  ```

### 3.2 Velocity Endpoint
`GET /api/projects/{project}/stats/velocity`
- **Response:**
  ```json
  [
    {"board_name": "Sprint 1", "committed": 12, "completed": 10},
    {"board_name": "Sprint 2", "committed": 15, "completed": 15}
  ]
  ```

## 4. Implementation Guidance for Developer

1. **New Controller:** `ProjectStatsController`.
2. **Library Installation:** `npm install recharts --legacy-peer-deps`.
3. **Frontend Implementation:**
   - Create `Pages/Projects/Reports.tsx`.
   - Use `recharts` to render the Burndown (LineChart) and Velocity (BarChart).
   - Ensure the colors match the Merkle CI: Cobalt Blue for Actual, Dash Red for Ideal.
