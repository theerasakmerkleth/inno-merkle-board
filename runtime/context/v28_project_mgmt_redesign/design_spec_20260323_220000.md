# UX/UI Design Specification: Simplified Project Management
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 28.0 (Management UX Simplification)
**Date:** 2026-03-23
**Designer:** Senior UX/UI Architect & Lead Designer

## 1. Problem Statement & Empathy Mapping
**User Pain Point:** *"function is complex... project setting board cart lane edit/lock user journey is not working for me"*
- **Current State:** Many configuration actions (lane renaming, reordering) are "hidden" behind double-clicks or hovering, making them non-discoverable. The Project Settings page feels like a separate, basic form rather than an integrated management hub.
- **Goal:** Redesign the management journey to be **Explicit, Discoverable, and Highly Branded**. Move from "Hidden Interactions" to a "Configuration Mode" paradigm. Simplify the Project Settings layout to feel like a premium Enterprise Command Center.

## 2. Information Architecture (IA) Redesign

### 2.1 The "Configuration Mode" Toggle (Kanban Board)
To reduce cognitive load during daily work while keeping power-user functions accessible:
- **Feature:** Add a **"Configure Board"** toggle switch in the Board Header.
- **Aesthetic:** A clean, Cobalt-bordered switch.
- **Behavior:**
    - **Work Mode (Default):** Board is for moving tasks. Lane drag-handles, "Add Column" buttons, and Delete icons are **HIDDEN**.
    - **Config Mode (ON):** The Board "unlocks". Lane drag-handles appear, titles show edit icons, and the "+ Add Column" zone becomes active. This makes the "User Journey" for editing the board structure explicit and intentional.

### 2.2 Project Settings Hub (`/projects/{key}/settings`)
- **Layout:** Move from a 2-column sidebar to a **Tabbed Header** or **Integrated Dashboard** layout.
- **Sections:**
    - **General:** Focused strictly on Name, Status, and Identity.
    - **Access & Team:** Large, clear data table of members with an prominent "Add Member" action. Use Avatars and clear Project Role labels.
    - **Danger Zone:** Isolated at the bottom with a clear warning background.

### 2.3 Explicit Visual Feedback (Locks & Permissions)
- **Feature:** If a user is a "Viewer" or lacks permissions to edit a project/board, display a **Lock Icon** 🔒 next to the project name in the sidebar and header.
- **Tooltip:** Hovering over locked elements provides a clear message: *"You do not have permission to manage this project. Contact your Admin."*

## 3. Interaction & Visual Language (Merkle CI)
- **Typography:** Use **Merkle Blue (`#040E4B`)** for section headers to denote authority.
- **Buttons:**
    - **Primary:** Cobalt (`#0328D1`) for "Save" and "Add".
    - **Secondary:** Outlined buttons for "Cancel" or "Settings".
- **Lanes:** When in "Config Mode", lanes receive a subtle blue dashed border to indicate they are in an "editable" state.
