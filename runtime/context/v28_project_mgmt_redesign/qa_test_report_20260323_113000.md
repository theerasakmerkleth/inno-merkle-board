# QA Test Report: Kanban DnD Cross-Lane Validation
**Project:** TaskFlow AI (Enterprise Task OS)
**Version:** 28.1 (DnD Regression Audit)
**Date:** 2026-03-23
**QA Engineer:** Senior QA Automation Engineer & SDET
**Status:** 🔴 NO-GO (Interaction Defect)

## 1. Executive Summary
Following the "Management UX Simplification" (v28.0), a regression has been reported in the core Kanban interaction: users are unable to reliably move task cards between different lanes (columns). The operation either fails to trigger a status update or the card "snaps back" to its original position without any feedback.

## 2. Defect Log

### 🔴 BUG-DnD-004: Intermittent Cross-Lane Drop Failure
- **Severity:** CRITICAL
- **Description:** Dragging a Task Card from Column A to Column B often fails to find a valid drop target, especially when dropping in the empty area of the column or when the column is already populated but the mouse is not directly over another card.
- **Root Cause Suspect:** The `closestCenter` collision detection algorithm is insufficient for nested sortable grids. It frequently miscalculates the target container when items are being moved rapidly between small vertical containers (Lanes).
- **Impact:** Fundamental loss of Kanban functionality. Users cannot progress work status.

### 🟠 BUG-DnD-005: Event Bubbling / Context Conflict
- **Severity:** MAJOR
- **Description:** Since there are two separate `DndContext` providers in `AppLayout` (for board tabs and lanes), there is a high risk of event interference. 
- **Recommendation:** Consolidate drag-and-drop logic or ensure strict separation of event listeners.

## 3. Recommended Technical Fixes
1.  **Collision Detection:** Switch from `closestCenter` to a custom combination of `pointerWithin` and `rectIntersection`. This ensures that the lane "under the mouse" is prioritized over the lane "closest to the center of the card".
2.  **Target Logic Refinement:** Ensure the `handleDragEnd` logic for tasks gracefully handles cases where `over.id` is a Column ID (dropping on empty space) vs a Task ID (dropping on a specific neighbor).

## 4. The Gatekeeper Decision
**Decision: 🔴 NO-GO**
The ability to move tasks between lanes is non-negotiable for a Task Management system. 

**Required Action:** Developer must implement a more robust collision detection strategy and verify the "Empty Lane Drop" scenario.
