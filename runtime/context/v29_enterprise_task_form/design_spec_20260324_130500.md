# UI/UX Design Specification: Enterprise-Grade Task Management Form (Jira-Style)

## 1. UX Architecture & Brand Alignment

**Design Philosophy:** The transition to an Enterprise-grade form requires a shift from a simple modal to a high-density, yet cognitively lightweight, slide-over panel (`<Sheet>`). We must balance the need for extensive metadata with a clean, uncrowded aesthetic. The user should be able to scan the core content (Title, Description) instantly, while secondary metadata (Dates, Points, Labels) rests cleanly in a structured sidebar.

**Layout Strategy: The Split-Pane Sheet**
To accommodate the new fields without vertical scrolling fatigue, the `<Sheet>` will be widened (e.g., `sm:w-[800px] md:w-[900px]`). Inside, we will implement a CSS Grid layout:
*   **Main Content Area (Left, ~65% width):** Title, TipTap Rich Text Editor, Attachments Dropzone, Activity/Comments.
*   **Metadata Sidebar (Right, ~35% width):** Status, Assignee, Reporter, Priority, Story Points, Start/Due Dates, Labels.

**Color & Typography Adherence (Merkle CI):**
*   **Backgrounds:** `bg-background` for the main sheet, `bg-muted/30` for the right-hand metadata sidebar to visually separate it.
*   **Borders:** Ultra-thin `border-border` dividers between metadata sections.
*   **Typography:** Strict use of tracking and uppercase for metadata labels (e.g., `text-[10px] font-bold uppercase tracking-widest text-muted-foreground`).
*   **Focus States:** All interactive elements (inputs, TipTap editor) must use the Merkle CI focus ring (`focus:ring-2 focus:ring-ring focus:ring-offset-2`).

## 2. Core Views UI Specification

### 2.1 The Enterprise Task Sheet (`<Sheet>`)
**Trigger:** Clicking "Create Card" or an existing Task Card.
**Animation:** `animate-in slide-in-from-right` (Standard shadcn Sheet).
**Width:** `w-full sm:w-[800px] md:w-[900px]` (Wider to support the split-pane).

#### Header (Sticky)
*   **Left:** Task ID (e.g., `CORE-142` in `font-mono text-muted-foreground`) + Task Title Input (Invisible border until hover/focus, large text `text-xl font-semibold text-foreground`).
*   **Right:** "Share" button (ghost), "..." menu (dropdown), and "Close" button.

#### Main Content Area (Left Column - `col-span-8` in a 12-col grid)
*   **Description (TipTap):** 
    *   **Toolbar:** Sticky at the top of the editor area. Minimalist square buttons (`p-1.5 rounded-sm hover:bg-muted`). Icons for Bold, Italic, Strike, H1, H2, Lists, Quote, Code.
    *   **Editor Body:** `min-h-[200px] prose prose-sm dark:prose-invert`. Subtle background `bg-background border border-border focus-within:border-ring`.
*   **Attachments:**
    *   **Dropzone:** A dashed area `border-2 border-dashed border-border rounded-lg bg-muted/10 p-6 flex flex-col items-center justify-center text-center hover:bg-muted/20 hover:border-primary/50 transition-colors`.
    *   **Icon:** Material Icon `cloud_upload` (large, muted).
    *   **Text:** "Drag & drop files or click to browse".
*   **Activity/Comments:** Stacked below attachments.

#### Metadata Sidebar (Right Column - `col-span-4` in a 12-col grid)
*   **Background:** `bg-muted/20` or simply separated by a left border `border-l border-border pl-6`.
*   **Section 1: Details (Grid)**
    *   **Status:** Custom select with status pills (e.g., `To Do` = gray pill, `In Progress` = blue pill, `Done` = green pill).
    *   **Assignee:** Select with User Avatar.
    *   **Reporter:** Read-only (Avatar + Name).
    *   **Priority:** Select with Icons (e.g., Red arrow up for Highest, Blue arrow down for Lowest).
    *   **Labels:** Multi-select input. Renders as distinct pills (`bg-accent text-accent-foreground text-[10px] px-2 py-0.5 rounded-full`).
    *   **Story Points:** Small number input `w-20`.
*   **Section 2: Dates (Grid)**
    *   **Start Date / Due Date:** Standard date pickers, stacked or side-by-side.

### 2.2 Responsive Behavior (Mobile)
*   On screens `< md` (Tablets/Phones), the 12-column grid collapses to a single column (`flex-col`). 
*   The Metadata Sidebar stacks *below* the Main Content Area, or is hidden behind a "Show Details" accordion/toggle to save space.

## 3. Interaction States & Edge Cases

*   **TipTap Focus:** When the editor is focused, the border color transitions to `border-primary` to clearly indicate active input.
*   **Attachment Drag-Over:** When a user drags a file over the Dropzone, the background changes to `bg-primary/5` and the border solidifies to `border-primary` to provide immediate drop kinetic feedback.
*   **Validation Error (Title):** If the title is empty upon saving, the input border turns `border-destructive`, and a subtle shake animation (`animate-shake`) is applied. An inline error `text-xs text-destructive` appears below.
*   **Loading State (Saving):** When `patch` or `post` is processing, the "Save" button text changes to a spinner, and `disabled:opacity-50` is applied.
