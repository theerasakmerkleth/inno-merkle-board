# UI/UX Design Specification: Export Roadmap/Board to Excel (Enterprise Jira-Style)

## 1. UX Architecture & Brand Alignment

**Design Philosophy:** The ability to export data to Excel is a critical enterprise feature. Stakeholders often need offline snapshots of project progress, capacity planning, and roadmaps. The interaction must be highly discoverable yet unobtrusive, adhering to the "Clean White" aesthetic. We will mirror the Enterprise Jira experience, which provides seamless, one-click exports and granular advanced exports without navigating away from the current context.

**Placement & Discoverability:**
*   **Location:** The export functionality will reside within the existing "Share / Actions (...)" menu or a dedicated `Export` button in the top-right header action bar of the Project, Board, and Roadmap views.
*   **Visual Prominence:** It should be a secondary action. Use a ghost button (`variant="ghost"`) or outline button with a clearly identifiable icon (e.g., `download` or `file_download`).

**Color & Typography Adherence (Merkle CI):**
*   **Icons:** Material Icons (`download`).
*   **Dropdown Menus:** Standard `shadcn/ui` Dropdown Menu with a white background (`bg-background`), subtle border (`border-border`), and hover states (`hover:bg-accent`).
*   **Feedback (Toasts):** When an export is initiated, a non-blocking toast notification (`bg-background` with `border-primary` accent) must appear to indicate progress, as generating large Excel files might take a few seconds.

## 2. Core Views UI Specification

### 2.1 The Export Action Menu (`<DropdownMenu>`)
**Trigger:** A button in the view header.
*   **Label:** "Export" or an icon-only button (`download` icon) for compact views.
*   **Menu Items:**
    *   `Export to Excel (Current View)`: Downloads what is currently filtered and visible on the screen.
    *   `Export All Tasks to Excel`: Downloads all tasks in the project/board regardless of current filters.
    *   `Export Roadmap (Timeline)`: Specifically for Roadmap views, exporting the timeline data.

### 2.2 Advanced Export Dialog (`<Dialog>`) - Optional/Future Scope
If the user needs to select specific columns or date ranges (Advanced Export):
**Trigger:** Clicking "Advanced Export..." from the Dropdown.
*   **Layout:** A standard `shadcn/ui` Dialog modal.
*   **Content:**
    *   **Field Selection:** A multi-select list or checklist of columns to include (e.g., Task ID, Title, Status, Assignee, Story Points, Start Date, Due Date, Labels).
    *   **Format Selection:** Radio buttons for `Excel (.xlsx)` vs `CSV (.csv)`.
*   **Actions:** "Cancel" (ghost), "Generate Report" (primary).

### 2.3 The Excel Output Structure (Reference)
The generated `.xlsx` file must look professional and be immediately usable for pivot tables or C-level reporting.
*   **Header Row:** Bold text, contrasting background color (e.g., Merkle Red or Cobalt Blue), frozen so it stays visible when scrolling.
*   **Data Columns:**
    *   Task Key (e.g., `CORE-123`)
    *   Title
    *   Description (PlainText, truncated if necessary)
    *   Status
    *   Priority
    *   Assignee
    *   Reporter
    *   Story Points
    *   Start Date
    *   Due Date
    *   Labels (Comma-separated)
    *   Checklist Progress (e.g., "3/5")
*   **Styling in Excel:** Auto-fit column widths where possible. Dates formatted correctly as Excel date objects, not just strings.

## 3. Interaction States & Edge Cases

*   **Loading State (Generation):** Generating an Excel file can take time.
    *   **Action:** When clicked, the menu closes, and a `<Toast>` appears in the bottom right: "Generating Excel report...".
    *   **Cursor:** The cursor does NOT change to `wait` globally to prevent blocking the user.
    *   **Completion:** The toast updates to "Report downloaded successfully!" (Success variant, `border-green-500` or Merkle CI equivalent) once the file download begins.
*   **Error State:** If the backend fails to generate the file (e.g., timeout, data too large).
    *   **Action:** A destructive `<Toast>` appears: "Failed to generate report. Please try again or contact support."
*   **Empty State:** If the current view has 0 tasks.
    *   **Action:** The export buttons should either be `disabled` (with `opacity-50` and `cursor-not-allowed`), or if clicked, instantly show an info toast: "No data available to export."

## 4. Frontend Developer Handoff Checklist

**shadcn/ui Components Required:**
*   [x] `DropdownMenu` (For the quick export options)
*   [x] `Button` (For the trigger)
*   [x] `Toast` / `Toaster` (CRITICAL: for async download feedback)
*   [ ] `Dialog` (If implementing Advanced Export)

**Technical Implementation Notes (React/Inertia):**
1.  **File Downloads in Inertia:** Inertia.js is designed for JSON responses, not binary file downloads. The Frontend Engineer MUST use a standard `window.location.href = '/export/endpoint'` or a native DOM `<a href="..." download>` hidden element, OR use a custom `fetch` call that handles the `Blob` response to trigger the download prompt.
2.  **Feedback Loop:** If using `window.location.href`, you cannot easily show a "Completed" toast. If a seamless UX is required, use `fetch()` to hit the export endpoint, wait for the `Blob`, show the "Generating..." toast, and then use `URL.createObjectURL(blob)` to trigger the download and show the "Completed" toast.
3.  **Placement:** Add the export trigger to `Project/Index.tsx` (Board view) and `Roadmap/Index.tsx` headers.

**Actionable Steps for Frontend:**
1.  Implement the Export Dropdown in the header action bars.
2.  Implement the async `Blob` fetch logic to handle the download gracefully while showing Toasts.
3.  Ensure the download file name is context-aware (e.g., `Merkle_Project_Board_2026-03-24.xlsx`).
