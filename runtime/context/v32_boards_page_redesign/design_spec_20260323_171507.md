# UI/UX Design Specification: Comprehensive Boards Page Redesign (Enterprise Grade)

## 1. UX Architecture & Brand Alignment

**Design Philosophy:** The Kanban Board is the beating heart of agile project management. Our goal is to transform the current board into a high-density, low-friction workspace that rivals Enterprise Jira. The design must be "Clean White," utilizing Merkle CI (Cobalt Blue for interactions, Merkle Red for high-priority alerts). We will eliminate heavy borders, rely on subtle shadows for depth, and use progressive disclosure to hide complexity until needed.

**Key Objectives:**
*   **Decluttering the Interface:** The board header should be minimalist. Actions like "Export" and "Configure Board" must be cleanly tucked away or visually deprioritized compared to "Create Task".
*   **Card Anatomy (Atomic Design):** Task cards must be highly scannable. We will use a dedicated grid layout inside the card to align avatars, priority icons, and task IDs perfectly.
*   **Kinetics & Feedback:** Drag-and-drop must feel weighty and precise. Drop zones must highlight clearly, and dragging a card should angle it slightly (tilt) to simulate picking it up.

## 2. Core Views UI Specification

### 2.1 The Global Board Header (Sticky)
*   **Background:** `bg-background/95 backdrop-blur supports-[backdrop-filter]:bg-background/60` (Glassmorphism effect so tasks scroll cleanly underneath).
*   **Left Side (Context):**
    *   Breadcrumbs: `Project Key` > `Board Name`.
    *   Primary Action: **Create Task** button (`bg-primary text-white hover:bg-primary/90`, solid).
*   **Right Side (Controls):**
    *   **View Toggles:** "Board", "Roadmap", "Reports" rendered as a cohesive `TabsList` style component or sleek segmented control, rather than disconnected buttons.
    *   **Actions Menu (`...`):** Consolidate "Configure Board" and "Export" into a single dropdown menu to save horizontal space.

### 2.2 Board Columns (Lanes)
*   **Container:** `bg-muted/10 border border-border/50 rounded-xl`. Subtle visual container, not a heavy box.
*   **Column Header:**
    *   Title: `text-xs font-semibold uppercase tracking-wider text-muted-foreground`.
    *   Task Count Badge: `bg-muted text-muted-foreground px-2 py-0.5 rounded-full text-[10px] font-mono`.
*   **Add Task (Inline):**
    *   Instead of a button that opens a full modal, the `+ Create Card` button at the bottom of the lane should ideally open an inline quick-create text input. If the user hits "Enter", it creates a skeleton task. If they click "More details", it opens the full Enterprise Sheet. *(MVP: Just ensure the button looks like a ghost button `hover:bg-muted/50 text-muted-foreground`)*.

### 2.3 Task Card Anatomy (High-Density)
*   **Card Container:** `bg-card border border-border shadow-sm hover:shadow-md transition-all rounded-lg p-3`.
*   **Top Row:**
    *   Task Key: `font-mono text-[10px] text-muted-foreground tracking-tight`.
    *   Align Right: Avatar of assignee (`h-5 w-5 rounded-full ring-1 ring-background`).
*   **Middle Row (Content):**
    *   Title: `text-sm font-medium leading-snug line-clamp-2` (Truncate at 2 lines).
*   **Bottom Row (Metadata):**
    *   Labels: Rendered as highly compact dots or max 1-2 pills (`text-[9px] px-1.5 py-0.5`).
    *   Priority Icon: Chevron Up (Red) for High, Equals (Gray) for Medium, Chevron Down (Blue) for Low.
    *   Story Points: `bg-muted text-muted-foreground px-1.5 py-0.5 rounded-sm text-[10px] font-mono`.
    *   Checklist Status: `text-[10px] text-muted-foreground flex items-center gap-1` (e.g., `☑ 2/5`).

## 3. Interaction States & Edge Cases

*   **Drag & Drop Kinetics (dnd-kit):**
    *   **Dragging (`isDragging`):** The original card turns into a subtle placeholder (`opacity-30 bg-muted border-dashed`). The dragged overlay card tilts by 3 degrees (`rotate-3 scale-105 shadow-xl cursor-grabbing`).
    *   **Dropzone (`isOver`):** The column being hovered over gains a subtle background highlight (`bg-primary/5 border-primary/20`).
*   **Empty States:**
    *   If a column has no tasks, display a dashed outline area with text: "Drop tasks here".
*   **Focus States (Accessibility):**
    *   Every interactive element (Cards, Buttons, Inputs) MUST have `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`.
