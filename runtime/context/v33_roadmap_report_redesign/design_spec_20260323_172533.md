# UI/UX Design Specification: Roadmap & Reports Page Redesign (Enterprise Grade)

## 1. UX Architecture & Brand Alignment

**Design Philosophy:** To ensure a cohesive, unified experience across the entire application, the "Strategic Roadmap" and "Agile Insights" (Reports) views must perfectly align with the newly redesigned Kanban Board. We will extend the "Clean White" aesthetic, eliminate disjointed navigation, and introduce the same segmented control paradigm for view switching. The goal is to make the entire suite feel like a single, seamless Enterprise application (Jira-style), rather than disconnected pages.

**Key Objectives:**
*   **Header Synchronization:** Replicate the sticky, glassmorphism header from the Kanban Board (`KanbanBoard.tsx`) onto both the Roadmap (`Roadmap.tsx`) and Reports (`Reports.tsx`) pages. This creates a unified "Global Toolbar" feel.
*   **Segmented Control Consistency:** The "Board | Roadmap | Reports" toggle must be the primary navigational anchor across all three views.
*   **Data Density & Whitespace:** Refine the Gantt chart (Roadmap) and Analytics dashboards (Reports) to use subtle borders (`border-border/50`), refined typography, and intelligent negative space, removing heavy visual "boxes".

## 2. Core Views UI Specification

### 2.1 The Global Page Header (Applied to BOTH Roadmap & Reports)
*   **Background:** `bg-background/95 backdrop-blur supports-[backdrop-filter]:bg-background/60 z-40`.
*   **Layout:** Consistent with the Board view.
    *   **Left:** Breadcrumbs (`Project Key > Name`) and primary contextual title (e.g., "Strategic Roadmap" or "Agile Insights" can be moved to the breadcrumb or a subtle sub-header if needed, but keeping the segmented control central is key).
    *   **Center/Right (Controls):** 
        *   **View Toggles:** The exact same segmented control `div` used in the Board view.
            *   In `Roadmap.tsx`, the "Roadmap" pill is the active state (`bg-background text-foreground shadow-sm`).
            *   In `Reports.tsx`, the "Reports" pill is the active state.
        *   **View-Specific Actions:**
            *   *Roadmap:* "Jump to Today" button and "Export" DropdownMenu.
            *   *Reports:* "Export" DropdownMenu (if applicable in the future) or date range pickers.

### 2.2 Strategic Roadmap (`Roadmap.tsx`) Redesign
*   **Gantt Chart Canvas:**
    *   **Background:** Clean `bg-background` with ultra-light vertical grid lines (`border-r border-border/30`) representing days/weeks.
    *   **Task Bars (The Timeline Cards):**
        *   Shape: `rounded-md shadow-sm`.
        *   Colors: Map to Priority (Merkle Red for High/Highest, Cobalt Blue for Medium, Slate for Low/Lowest). Alternatively, use `bg-primary/90` for a unified look.
        *   Content: Task Title (`text-xs text-white font-medium truncate px-2 py-1`).
    *   **Sidebar (Task List):**
        *   Clean, borderless list (`border-r border-border/50`).
        *   Task rows should hover with `bg-muted/50`.
        *   Display Task ID (mono) and Title (truncated).
    *   **Dependencies (Lines):** Render dependency lines using subtle SVG paths (`stroke-muted-foreground/40 stroke-2`).

### 2.3 Agile Insights / Reports (`Reports.tsx`) Redesign
*   **Dashboard Layout:**
    *   Use a CSS Grid (`grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-6`).
*   **Metric Cards (Atomic Design):**
    *   **Container:** `bg-card border border-border/50 rounded-xl p-6 shadow-sm hover:shadow-md transition-shadow`.
    *   **Anatomy:**
        *   Top Left: Title (`text-xs uppercase tracking-wider text-muted-foreground font-semibold`).
        *   Top Right: Minimalist Material Icon representing the metric (e.g., `trending_up`).
        *   Center: Large, bold metric value (`text-3xl font-bold text-foreground`).
        *   Bottom: Delta or context (`text-[10px] text-muted-foreground`, e.g., "+2% from last sprint").
*   **Charts (e.g., Burn-down, Velocity):**
    *   Use Recharts or Chart.js (if integrated).
    *   Strip away all chart borders and grid lines except for the essential X/Y axes (`stroke="hsl(var(--border))"`).
    *   Use Merkle CI colors for lines/bars (Primary: Cobalt, Secondary: Muted).

## 3. Interaction States & Edge Cases

*   **Loading States:**
    *   Instead of spinners, use Skeleton loaders (`<Skeleton className="h-4 w-[200px]" />`) for the charts and metric cards to prevent layout shifts.
*   **Empty States:**
    *   **Roadmap:** If no tasks have dates, show an elegant empty state placeholder: A subtle illustration or icon (`calendar_today`) with text: "Your roadmap is empty. Add start and due dates to tasks to visualize them here."
    *   **Reports:** If no data exists, show: "Not enough data to generate insights yet."
