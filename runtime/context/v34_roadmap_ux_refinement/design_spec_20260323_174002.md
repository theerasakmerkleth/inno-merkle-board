# UI/UX Design Specification: Strategic Roadmap UX & Legibility Refinement

## 1. UX Architecture & Brand Alignment

**Design Philosophy:** The Strategic Roadmap must provide an at-a-glance understanding of macro-level timelines without causing eye strain. The previous iteration introduced the correct structure, but we must urgently refine the **contrast ratios, background layers, and typography legibility** to ensure a premium Enterprise User Experience (Jira-Style). 

**Key Objectives:**
*   **Prevent Visual Bleed-through:** Ensure that sticky headers and the left-hand task sidebar have solid, opaque backgrounds (`bg-background` or `bg-card`) so that scrolling timeline bars do not awkwardly show underneath them.
*   **High-Contrast Task Bars:** The Gantt chart task bars MUST have perfectly legible text. We will mandate `text-white` with a crisp `drop-shadow-md` for all colored priority bars (Merkle Red, Cobalt Blue) to guarantee WCAG 2.1 AA compliance.
*   **Grid Clarity:** Weekends should be distinctly, but softly, shaded (`bg-muted/30`) to separate them from working days without overpowering the actual task bars. The "Today" indicator must pop.

## 2. Core Views UI Specification

### 2.1 The Sticky Task Sidebar (Left Panel)
*   **Background & Z-Index:** MUST be solid `bg-background` (not transparent) with `z-20`. Add a right-side shadow (`shadow-[2px_0_10px_-4px_rgba(0,0,0,0.15)]`) to create a clear physical elevation above the scrolling timeline grid.
*   **Row Styling:** 
    *   Default: `bg-background border-b border-border/40`.
    *   Hover State: `hover:bg-muted/50 transition-colors cursor-pointer`.
*   **Typography:**
    *   Task ID: `text-[10px] font-mono text-muted-foreground bg-muted/50 px-1.5 py-0.5 rounded-sm`.
    *   Task Title: `text-xs text-foreground font-medium truncate`. On row hover, the title text should NOT change color dramatically; keep it `text-foreground` to maintain a professional look, perhaps only slightly darkening or staying stable.

### 2.2 The Sticky Timeline Header (Top Panel)
*   **Background & Z-Index:** MUST be `bg-background/95 backdrop-blur-md supports-[backdrop-filter]:bg-background/80` with `z-20` and a solid bottom border `border-b border-border`.
*   **Typography (Months/Days):**
    *   Months: `text-[10px] font-bold uppercase tracking-widest text-muted-foreground`.
    *   Days: `text-[9px] text-muted-foreground`.
    *   Today's Date: Highlighted aggressively `text-primary font-extrabold bg-primary/10`.

### 2.3 The Gantt Canvas & Task Bars
*   **Weekend Columns:** `bg-muted/20`.
*   **Today's Column:** `bg-primary/5` with a continuous vertical 1px line `bg-primary/50` running down the entire chart.
*   **Task Bar Anatomy:**
    *   **Container:** `h-6 rounded-md shadow-sm border border-black/5` (The tiny border helps define the edge against similar background colors).
    *   **Colors (Priority Mapped):** 
        *   Highest/High: `bg-destructive` (Merkle Red).
        *   Medium: `bg-primary` (Cobalt Blue).
        *   Low/Lowest: `bg-slate-500` (Neutral Gray).
        *   Done Status: `bg-muted text-muted-foreground` (Overrides priority).
    *   **Typography (Crucial Fix):** The text inside the active colored bars MUST be `text-[10px] font-medium text-white drop-shadow-md truncate px-2`. The drop-shadow ensures legibility even if the user applies a custom, lighter theme color in the future.
    *   **Hover State:** `hover:ring-2 hover:ring-primary/50 hover:brightness-110 transition-all cursor-pointer`.

## 3. Interaction States & Edge Cases

*   **Scrolling Matrix:** When scrolling horizontally, the left sidebar MUST remain perfectly static and opaque. When scrolling vertically, the top date header MUST remain perfectly static and opaque.
*   **Empty Dates (Warning State):** Tasks without `start_date` or `due_date` cannot be drawn on the canvas. The warning box below the sidebar must use a soft warning palette: `bg-amber-500/10 border-y border-amber-500/20 text-amber-700` with a clear `warning_amber` icon.
