# UI/UX Design Specification: Agile Insights (Reports) Export Capabilities

## 1. UX Architecture & Brand Alignment

**Design Philosophy:** The Agile Insights dashboard must not only display data elegantly but also allow seamless extraction. We are extending the "Clean White" Enterprise aesthetic to include robust Export functionalities (PDF & Excel) directly from the Reports view. The user experience must perfectly mirror the Kanban Board and Roadmap export behaviors—utilizing a secondary `DropdownMenu` in the global header and providing non-blocking asynchronous feedback (`toast`).

**Key Objectives:**
*   **Unified Export Menu:** Introduce the standard `...` or `Export` button in the Reports header, aligning it right next to the segmented control or breadcrumbs to maintain visual consistency across all three primary views.
*   **Clear Export Options:** Users must easily distinguish between exporting the visual dashboard (PDF) and the raw metric data (Excel).
*   **Asynchronous UX Loop:** Ensure the `Generating PDF...` or `Generating Excel...` toast notifications appear instantly, preventing user confusion while the potentially heavy rendering process occurs in the background.

## 2. Core Views UI Specification

### 2.1 The Reports Global Header (`Reports.tsx`)
*   **Layout Adjustments:** The current header already features the Breadcrumbs and Segmented Control. We must add the Export `DropdownMenu` to the far right of the header controls area.
*   **Export Button (Trigger):**
    *   Style: Ghost button (`variant="ghost"`) or an outlined button (`border border-border bg-transparent text-muted-foreground hover:text-foreground hover:bg-muted`).
    *   Content: Material Icon (`download`) + Text ("Export").
*   **Dropdown Menu Items:**
    *   `Export Dashboard (PDF)`: Icon `picture_as_pdf`. Exports the entire visible screen area (Charts, Metrics).
    *   `Export Burndown Data (Excel)`: Icon `table_view`. Exports the raw `remaining` vs `ideal` data points.
    *   `Export Velocity Data (Excel)`: Icon `table_view`. Exports the raw `committed` vs `completed` data points.

### 2.2 The PDF Export Output
*   **Visual Fidelity:** The exported PDF MUST resemble the on-screen dashboard as closely as possible. It should retain the `rounded-xl` card borders, the chart colors (Merkle Red, Cobalt Blue), and the typography.
*   **Orientation:** Landscape is generally preferred for dashboards to prevent charts from being squashed.
*   **Header/Footer (Optional but Recommended):** The PDF should ideally include a header indicating the Project Name, "Agile Insights Report", and the generation date.

### 2.3 The Excel Export Output
*   **Data Structure:** The Excel files must be clean, structured tables.
    *   *Burndown:* Columns -> Date, Ideal Work Remaining, Actual Work Remaining.
    *   *Velocity:* Columns -> Sprint/Board Name, Committed Points, Completed Points.
*   **Styling:** Follow the Jira-style established in `TasksExport.php`. Use a frozen header row with bold white text on a Merkle Red (`#DD3039`) or Cobalt Blue background.

## 3. Interaction States & Edge Cases

*   **Generating PDF Feedback:** Client-side PDF generation (e.g., using `html2canvas` + `jspdf` or `react-to-pdf`) can briefly freeze the main thread.
    *   **Crucial State:** The `toast.loading('Generating PDF...')` MUST be fired *before* the synchronous rendering block starts, ensuring the user gets immediate feedback.
*   **Empty Charts:** If the burndown or velocity data arrays are empty, the export buttons should either be disabled or, if clicked, download a document that clearly states "No data available for this report."
