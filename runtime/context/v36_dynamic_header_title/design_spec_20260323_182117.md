# UI/UX Design Specification: Dynamic Page Header Titles (Merkle CI Alignment)

## 1. UX Architecture & Brand Alignment

**Design Philosophy:** Context is paramount in complex enterprise tools. The header must act as a reliable anchor, clearly stating the user's current location within the project. We will standardize the "Primary Title Block" across all three views (Board, Roadmap, Reports) to create a persistent visual hierarchy.

**Placement:** The Title Block (comprising a main `<h1>` and a subtle `<p>` description) will be placed on the left side of the global header, replacing or augmenting the existing contextual buttons.

**Merkle CI Alignment:**
*   **H1 Style:** `text-lg font-semibold text-foreground` (Consistent with Merkle's bold, professional aesthetic).
*   **P Style:** `text-[11px] text-muted-foreground` (Low-emphasis secondary information).
*   **Whitespace:** Maintain `gap-4` between the Title Block and any adjacent action buttons (e.g., "Create Task").

## 2. Dynamic Content Mapping

The UI MUST render the following content based on the active route:

| View Name | Primary Title (`<h1>`) | Secondary Subtitle (`<p>`) |
| :--- | :--- | :--- |
| **Kanban Board** | `{active_board.name}` | "Manage tasks and team workflow." |
| **Strategic Roadmap** | "Strategic Roadmap" | "Macro-level timeline and blockers." |
| **Agile Insights** | "Agile Insights" | "Measure team velocity and sprint health." |

## 3. UI Component Specification (React Refactoring)

### 2.1 Kanban Board Layout Update (`KanbanBoard.tsx`)
*   **Current State:** Shows the "Create Task" button as the leftmost element.
*   **Redesign:** 
    *   Insert the Title Block to the *left* of the "Create Task" button.
    *   If the `active_board.name` is very long, use `truncate max-w-[200px]` on the `<h1>`.
    *   Ensure the "Create Task" button maintains its visual prominence but sits logically after the page context.

### 2.2 Consistency Check (`Roadmap.tsx` & `Reports.tsx`)
*   Ensure the font sizes, weights, and colors match the Board's Title Block exactly.
*   Ensure the breadcrumbs in `AppLayout` continue to show `[Project Key] Project Name` to provide the global project context while the header title provides the view context.

## 4. Interaction States

*   **Responsive Collapsing:** On small screens (`< sm`), the subtitle `<p>` should be hidden (`hidden sm:block`) to prevent vertical header bloat. The `<h1>` size may be reduced to `text-base`.
*   **Navigation Transition:** As the user clicks the Segmented Control to switch views, the title should update instantly (Inertia handles the state refresh, providing a seamless transition).
