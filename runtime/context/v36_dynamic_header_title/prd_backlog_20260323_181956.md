# Product Requirements Document (PRD): Dynamic Page Header Titles

## 1. Value Proposition
**Goal:** To dynamically update the main title in the global header to reflect the current active view (Board, Roadmap, Reports) that the user has navigated to.
**ROI:** Improves contextual awareness and navigation clarity, especially as the application scales and users switch rapidly between different data visualizations (Kanban vs Gantt vs Dashboards). A clear, descriptive page title prevents users from getting "lost" within the Single Page Application (SPA) experience.

## 2. User Empathy & Journeys
**Persona:** All Users (Team Members, Project Managers, Stakeholders).
**Journey:** A Project Manager is analyzing a project. They click on the "Roadmap" tab in the segmented control. The page content transitions smoothly, and simultaneously, the large title in the top-left of the header changes from the Project Name (or previous title) to explicitly state "Roadmap". They then click "Reports", and the title instantly updates to "Reports" (or "Agile Insights"), confirming their location in the system.

## 3. Epics & User Stories

### Epic 1: Contextual Header Typography

#### US-1.1: Dynamic View Titles
**Story:** *As a User, I want the main header title to clearly display the name of the page I am currently viewing, so that I always know my context within the project.*

* **AC 1 (Board View):** Given the user navigates to the Kanban Board (`/projects/{key}/boards`), When the page renders, Then the primary `<h1>` title in the header MUST display the name of the currently active board (e.g., "Main Board" or "Sprint 1") or "Kanban Board" if a specific board context isn't the primary focus. *Note: Currently, the Board view might be displaying the Project Name. This should be refined to show the Board Name or View Name.*
* **AC 2 (Roadmap View):** Given the user navigates to the Roadmap (`/projects/{key}/roadmap`), When the page renders, Then the primary `<h1>` title MUST explicitly read "Strategic Roadmap" (or simply "Roadmap").
* **AC 3 (Reports View):** Given the user navigates to the Reports page (`/projects/{key}/reports`), When the page renders, Then the primary `<h1>` title MUST explicitly read "Agile Insights" (or simply "Reports").
* **AC 4 (Subtitle Context):** The `<p>` subtitle beneath the main `<h1>` should also dynamically update to provide a brief description of the current view (e.g., "Manage tasks and workflow" for Board, "Macro-level timeline" for Roadmap).

## 4. Edge Cases & Failure Modes
* **Breadcrumb Redundancy:** If the main title displays the view name (e.g., "Roadmap"), ensure the breadcrumbs (`[Project Key] Project Name`) are still visible so the user doesn't lose their project-level context.
* **Long Board Names:** In the Board view, if the active board has a very long name, the `<h1>` title must truncate gracefully with an ellipsis (`...`) to prevent pushing the Segmented Control off the screen on smaller devices.

## 5. Non-Functional Requirements (NFRs)
* **Consistency:** The typography (font size, weight, color) of the dynamic title must be identical across all three views (`KanbanBoard.tsx`, `Roadmap.tsx`, `Reports.tsx`) to maintain the "Global Header" illusion.
* **Accessibility:** The dynamic `<h1>` must be announced correctly by screen readers when the route changes.
