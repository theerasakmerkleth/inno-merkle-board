# Product Requirements Document (PRD): Dynamic HTML Document Titles

## 1. Value Proposition
**Goal:** To dynamically update the HTML `<title>` tag of the document to reflect the current page name, improving browser tab management, accessibility, and SEO.
**ROI:** Users often have multiple tabs open. Descriptive titles (e.g., "Sprint 1 | CORE") allow users to switch context efficiently and understand exactly which board or report they are viewing without clicking into the tab.

## 2. User Empathy & Journeys
**Persona:** All Users.
**Journey:** A user is managing three different projects. They have several tabs open: one for the Board, one for the Roadmap, and one for the Reports. Without dynamic titles, all tabs might just say "Merkle Board". With this feature, they can see "Board", "Roadmap", and "Agile Insights" directly in the browser tab bar, allowing for instant context switching.

## 3. Epics & User Stories

### Epic 1: Document Metadata Management

#### US-1.1: Inertia Head Integration
**Story:** *As a User, I want the browser tab title to match the page I am currently viewing, so that I can easily identify the tab.*

* **AC 1 (Board View):** Given the user is on a Board page, When the page renders, Then the HTML title MUST be set to `{active_board.name} | {project.key}`.
* **AC 2 (Roadmap View):** Given the user is on the Roadmap page, When the page renders, Then the HTML title MUST be set to `Roadmap | {project.key}`.
* **AC 3 (Reports View):** Given the user is on the Reports page, When the page renders, Then the HTML title MUST be set to `Agile Insights | {project.key}`.
* **AC 4 (Settings View):** Given the user is on the Project Settings page, When the page renders, Then the HTML title MUST be set to `Settings | {project.key}`.

## 4. Technical Implementation Direction
*   **Frontend:** The Frontend Engineer MUST use the `<Head />` component from `@inertiajs/react` in each top-level page component.
*   **Global App Name:** If a global app name is defined (e.g., "Merkle AI"), it should be appended as a suffix (e.g., "Roadmap | CORE | Merkle AI").

## 5. Non-Functional Requirements (NFRs)
*   **Accessibility:** The title update should be seamless and triggered by the client-side router (Inertia).
*   **Consistency:** The pattern `{Page} | {Context}` must be strictly followed across all project-level views.
