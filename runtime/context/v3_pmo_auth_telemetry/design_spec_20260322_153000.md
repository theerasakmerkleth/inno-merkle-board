## 2. Component Verification

### 2.1 Authentication Flow (`/login`)
*   **Audit Status:** 🟢 **PASS**
*   **Visuals:** The login page accurately reflects the Merkle enterprise theme. The `bg-slate-950` background provides a modern, dark-mode aesthetic, while the primary action button (`Sign in`) correctly uses the Deep Red accent (`bg-red-700 hover:bg-red-800`).
*   **Typography & Icons:** The `view_kanban` Material Icon and Inter/Geist typography create a professional entry point.
*   **UX:** Error handling (e.g., invalid credentials, disabled accounts) is cleanly integrated below the input fields, ensuring users understand why access might be denied.

### 2.2 Kanban Board RBAC Integration (`/`)
*   **Audit Status:** 🟢 **PASS**
*   **Header Updates:** The top navigation bar now elegantly displays the logged-in user's name alongside a standard `Logout` button, clarifying the current session context.
*   **Visual Locking (Jira-style):** The Developer successfully implemented the visual cues requested in v3.0. 
    *   Columns restricted by the user's `can_move_to_done` permission correctly display a subtle `lock` icon.
    *   The restricted drop zone is highlighted with a warning styling (`bg-red-950/10 border-red-900/30`), providing immediate visual feedback *before* a user attempts an invalid action.
    *   The browser `alert()` serves as a functional fallback, though in future sprints we may upgrade this to a `shadcn/Toast` notification.
