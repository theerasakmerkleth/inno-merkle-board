# UI/UX Design Specification: Board Activity Stream (v48)

## 1. UX Architecture & Strategy

**Design Philosophy: "Collective Awareness"**
In a high-density enterprise environment, knowing "what's happening now" across the entire board is crucial for PMs and team leads. Instead of hiding history inside individual task modals, we are introducing a **Board Activity Stream**. This provides a macro-view of the workflow without requiring users to hunt for updates.

**Key Features:**
*   **Slide-over Activity Feed:** A dedicated `<Sheet>` that slides in from the right, displaying all recent actions performed on the current board.
*   **Contextual Linking:** Every history item links directly to the relevant task modal.
*   **Compact Timeline:** A vertical, non-intrusive timeline that prioritizes the "Who", "What", and "When".

## 3. Core Views UI Specification

### 3.1 Activity Trigger (Board Header)
*   **Icon:** `history` (Material Icons).
*   **Location:** Top right of the Kanban Board, next to the "Configure Board" dropdown.
*   **Style:** `hover:bg-zinc-100 p-2 rounded-lg transition-all`.

### 3.2 Activity Stream Item (Polymorphic)
*   **Standard Move:** `[Avatar] **User** moved **Task ID** from **Column A** to **Column B**`.
*   **Creation:** `[Avatar] **User** created task **Task ID**`.
*   **Comment:** `[Avatar] **User** commented on **Task ID**: "..."`.
*   **Interaction:** Clicking the Task ID or the card area triggers `openEditModal(task)`.

### 3.3 The "Empty State"
*   If no activity exists within the last 7 days, show a minimal "No recent activity" message with a grayscale `history` icon.

## 4. Interaction States & Kinetics
*   **Sheet Animation:** Standard shadcn `slide-in-from-right` with a duration of `300ms`.
*   **Item Hover:** `hover:bg-zinc-50/80 cursor-pointer`.
*   **Live Updates:** Use a subtle pulse animation on the "Activity" icon in the header if a new event occurs while the panel is closed.
