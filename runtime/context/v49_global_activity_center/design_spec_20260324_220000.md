# UI/UX Design Specification: Global Activity Center (v49)

## 1. UX Architecture & Strategy

**Design Philosophy: "Personalized Traceability"**
Moving from a localized board history to a **Global Activity Center** transforms how users stay informed. Instead of checking each board, users now have a single source of truth for all updates relevant to them. This reduces "tab-switching" fatigue and ensures no critical update is missed.

**Key Changes:**
*   **Deprecation:** Remove the board-specific "History" trigger from `KanbanBoard.tsx`.
*   **New Navigation:** Add "Activity Log" to the Primary Sidebar in `AppLayout.tsx`.
*   **Centralized Feed:** A dedicated full-page view that aggregates logs across all projects and boards, filtered by the user's involvement.

## 3. Core Views UI Specification

### 3.1 Sidebar Navigation (`AppLayout.tsx`)
*   **Label:** "Activity Log"
*   **Position:** Below "Dashboard" and above "User Management".
*   **Icon:** `history`.

### 3.2 Global Activity Center (`Activity/Index.tsx`)
*   **Header:** Title "Activity Log" with a subtitle "Tracking updates across your workspace".
*   **Filter Bar:** 
    *   *Search:* Search by Task ID or Title.
    *   *Project Filter:* Dropdown to scope logs to a specific project.
    *   *Action Type:* Filter by "Comments", "Status Changes", or "Assignments".
*   **The Timeline Feed:**
    *   **Actor:** User avatar and name (Bold).
    *   **Action:** Verb-based description (e.g., "moved", "commented on").
    *   **Subject:** Linked Task Title and ID.
    *   **Timestamp:** Relative time (e.g., "2 hours ago") aligned to the right.

### 3.3 Logic: "Relate to Me"
The feed should prioritize and default to logs where:
1.  The user is the **Assignee** of the task.
2.  The user is a **Member** of the project where the activity occurred.
3.  The user was **Mentioned** or is part of the comment thread.

## 4. Interaction States
*   **Infinite Scroll:** As users scroll down, more logs are fetched (Standard Pagination).
*   **Empty State:** A clean illustration with the text "Your workspace is quiet. No recent activity found."
*   **Navigation:** Clicking any activity item redirects the user to the specific board and automatically opens the Task Modal for that task.

## 5. Frontend & Backend Handoff

**Backend Actions:**
*   Update `ActivityLogController` to support a global index with advanced filtering (by project, by involvement).
*   Ensure the response includes `task` and `project` relationships for context badges.

**Frontend Actions:**

*   Create `resources/js/Pages/Activity/Index.tsx`.
