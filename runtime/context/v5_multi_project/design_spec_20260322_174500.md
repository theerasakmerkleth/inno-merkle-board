**Status:** Ready for Architect & Developer Implementation
**Reference:** `prd_backlog_20260322_173000.md`

---

## 1. UX/UI Vision Alignment

To support the new **Multi-Project & Board Architecture**, we must evolve the UI from a single-page app model to a structured, hierarchical dashboard. We will maintain the "Swiss Minimalist" design language (flat surfaces, subtle borders, high data density) introduced in v5.0.

The new hierarchy is: **Workspace -> Projects -> Boards -> Columns -> Tasks**.

---

## 2. Structural Redesign (Wireframes)

### 2.1 Global Application Shell
We are moving away from a simple top-header to a layout that supports deeper navigation.

*   **Left Sidebar (Project Switcher):** 
    *   A narrow, collapsible sidebar (`w-64`) on the left. 
    *   **Top Area:** A minimalist `<Select>` or custom Dropdown Menu serving as the **Project Switcher**. Displays the current Project Key (e.g., `[CORE] Core Backend`) and allows switching between accessible projects.
    *   **Middle Area:** Navigation links for the current Project: `Boards`, `Team Settings` (RBAC), `Settings`.
    *   **Bottom Area:** User Profile & Logout (moved from the top-header).
    *   **Visuals:** `bg-zinc-950` with a subtle right border `border-r border-zinc-900/50`.

*   **Main Content Area:** 
    *   Occupies the remaining width.
    *   **Top Header:** Displays the context of the current view (e.g., the name of the active Board).

### 2.2 Board Navigation (`/projects/{project}/boards`)
*   **Board Tabs:** Since a project can have multiple boards, we will use minimal Shadcn `<Tabs>` right below the Main Content Header to allow quick switching between boards (e.g., "Sprint 22", "Backlog").
*   **Visuals:** Active tab has a subtle bottom border (`border-b-2 border-red-700`). Inactive tabs are muted `text-zinc-500`.

### 2.3 Updated Kanban & Task Cards
*   **Task IDs:** The Jira-style Project Key is critical. Every Task Card must display its ID prominently in the top-left corner using a monospace font (e.g., `CORE-142` instead of just `#142`). 
*   **Component Update:** Modify `TaskCard` in `KanbanBoard.tsx` to accept and render a `projectKey` string alongside the task ID.

---

## 3. Component Interaction Flow

1.  **Project Context:** When the user logs in, they are redirected to their most recently accessed Project, or a default `/projects` listing page if they have none selected.
2.  **Board Context:** Selecting a Project loads its default Board via Inertia.
3.  **Creation:** 
    *   Sidebar contains a subtle `+ New Project` button (visible only to Global Admins/PMs).
    *   Board Tabs area contains a `+ Create Board` ghost button.

---

## 4. Handover to Solution Architect
The UI now requires a strictly hierarchical data feed.
1.  **API Requirement:** The Inertia response payload for the Kanban page MUST include:
    *   `current_project`: { id, key, name }
    *   `available_projects`: [] (for the switcher dropdown)
    *   `boards`: [] (for the tabs)
    *   `active_board`: { columns, tasks }
2.  **Architectural Need:** Please define the database changes required to generate the `project.key` (e.g., 'CORE') and link tasks through the new `boards` table.
