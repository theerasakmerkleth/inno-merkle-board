## 1. Database Schema Additions

### 1.1 The `comments` Table (NEW)
To support in-task collaboration (US-018), a new table is required.
*   `id` (PK)
*   `task_id` (FK -> tasks.id, Cascade Delete)
*   `user_id` (FK -> users.id, Cascade Delete) - The author.
*   `content` (Text)
*   `timestamps` (created_at is crucial for the timeline view)

## 2. Backend Logic & Application Layer

### 2.1 My Tasks API (Global Context)
*   **Route:** `GET /` (or `/my-tasks`)
*   **Controller Logic:**
    ```php
    $tasks = Task::where('assignee_id', auth()->id())
                 ->whereNotIn('status', ['done']) // Focus on actionable items
                 ->with(['project', 'board'])
                 ->orderByRaw("FIELD(priority, 'high', 'medium', 'low')")
                 ->get();
    ```

### 2.2 Enterprise Analytics API
*   **Route:** `GET /analytics`
*   **Controller Logic (`DashboardController`):**
    *   **Resource Load:** Group tasks by `assignee_id` where `status` != 'done'. Join with `users` to get names. Count totals.
    *   **Cycle Time:** Query `task_transitions` where `to_status` = 'done'. Join the same task's `from_status` = 'in_progress' (or earliest transition). Calculate the average time difference in hours/days.

### 2.3 Comments API
*   **Route:** `POST /tasks/{task}/comments`
*   **Controller Logic:** Validate content. Create comment.
*   **Policy (`CommentPolicy`):** Users must have `Manager` or `Contributor` role in the task's parent project to post. Viewers can only read (which is loaded when fetching the task details).
